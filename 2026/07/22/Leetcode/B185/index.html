3963 - Create Grid With Exactly One Path

题意

题目链接：Create Grid With Exactly One Path

给定网格的行数 m 和列数 n，需要构造一个只包含 '.' 和 '#' 的 m \times n 网格：

  • '.' 表示可以经过的空地；
  • '#' 表示不能经过的障碍。

路径从左上角 (0,0) 出发，只能向右或向下移动，最终到达右下角 (m-1,n-1)。要求构造出的网格中恰好只有一条合法路径。

  • 1 <= m, n <= 25

解题思路

题目允许返回任意合法构造，因此只需要预先选定一条简单路径，再把其他位置全部设为障碍。

最直接的选择是：

  1. 沿第一行一直向右，从 (0,0) 走到 (0,n-1)；
  2. 沿最后一列一直向下，走到 (m-1,n-1)。

因此，先把整个网格初始化为 '#'，再将第一行和最后一列全部改为 '.'。

这条路径一定存在。下面说明它为什么是唯一的：

  • 在第一行中，除了继续向右以外，向下的位置都是障碍；只有到达最后一列后才能向下；
  • 在最后一列中已经不能继续向右，只能不断向下直到终点；
  • 其余位置全部是障碍，不可能形成分支或绕行路径。

当 m=1 或 n=1 时，构造会退化成唯一的一行或一列，结论仍然成立。

复杂度分析

构造并返回整个网格需要访问 m \times n 个位置。

时间复杂度：O(mn)。

空间复杂度：O(mn)，用于保存返回的网格；除返回值外的额外空间复杂度为 O(1)。

变形

如果要求路径经过某个指定位置 (x,y)，可以先向右走到第 y 列，再向下走到第 x 行，然后继续选取一条不产生分支的折线路径到达终点。

题目链接：Minimum Lights to Illuminate a Road

给定长度为 n 的数组 lights，表示道路上的 n 个位置。

若 lights[i]=0，则该位置没有正常工作的灯。

可以在任意位置安装额外的灯，每盏额外的灯可以照亮自身及相邻位置，即半径为 1。求使整条道路都被照亮所需的最少额外灯数。

  • 1 <= n <= 10^5

问题分成两个阶段：

1. DP 状态

将 x 转换成十进制数位数组 dig，从高位到低位填写数字。

dfs(pos,pre,lim) 表示从第 pos 位开始填写时的合法方案数：

  • pos：当前处理到的数位下标；
  • pre：前一个有效数位，取值为 0 到 9；
  • pre=10：尚未填写第一个非零数位，即当前仍处于前导零阶段；
  • lim：当前前缀是否与 x 的前缀完全相同。

若 lim=true，当前位最多只能填写 dig[pos]；否则可以填写 0 到 9。

2. 前导零的处理

前导零不属于整数的十进制表示，不能参与相邻数位差的判断。
