## 3936 - Minimum Swaps to Move Zeros to End

**题意**

题目链接：[Minimum Swaps to Move Zeros to End](https://leetcode.com/problems/minimum-swaps-to-move-zeros-to-end/description/)

给定一个整数数组 `nums`。每次操作可以选择两个不同位置 `i` 和 `j`，交换 `nums[i]` 与 `nums[j]`。要求用最少操作次数，使得数组中的所有 `0` 都移动到末尾。

**解题思路**

设数组中 `0` 的数量为 `z`。最终数组的最后 `z` 个位置必须都是 `0`，因此只需要观察前 `n - z` 个位置。

这 `n - z` 个位置中每出现一个 `0`，就意味着它必须与后缀中的一个非零数交换一次。一次交换可以同时修正一个前缀 `0` 和一个后缀非零数，所以最少交换次数就是前 `n - z` 个位置中 `0` 的数量。

**代码**

```cpp
class Solution {
public:
    int minimumSwaps(vector<int>& nums) {
        int n = nums.size();
        int zeroCount = 0;

        for (int x : nums) {
            if (x == 0) {
                zeroCount++;
            }
        }

        int ans = 0;
        for (int i = 0; i < n - zeroCount; i++) {
            if (nums[i] == 0) {
                ans++;
            }
        }

        return ans;
    }
};
```

**复杂度分析**

时间复杂度：$O(n)$，需要遍历数组统计零的数量，再扫描一次前缀。

空间复杂度：$O(1)$，只使用了常数个额外变量。

## 3937 - Minimum Operations to Make Array Modulo Alternating I

**题意**

题目链接：[Minimum Operations to Make Array Modulo Alternating I](https://leetcode.com/problems/minimum-operations-to-make-array-modulo-alternating-i/description/)

给定数组 `nums` 和整数 `k`。每次操作可以让任意一个元素加一或减一。要求存在两个不同的整数 `x` 和 `y`，满足：

- 所有偶数下标 `i`：`nums[i] % k == x`
- 所有奇数下标 `i`：`nums[i] % k == y`

返回最少操作次数。

**解题思路**

先看单个元素。若当前余数为 `r`，目标余数为 `t`，那么最少操作次数就是模 `k` 环上的最短距离：

```text
min((r - t + k) % k, (t - r + k) % k)
```

因为每次只能加一或减一，所以余数会沿着模环顺时针或逆时针移动，取较短的一条即可。

由于 `k <= 100`，可以直接枚举偶数下标统一变成的余数 `x`，以及奇数下标统一变成的余数 `y`，并要求 `x != y`。

先分别预处理：

- `evenCost[x]`：所有偶数下标变成余数 `x` 的总代价
- `oddCost[y]`：所有奇数下标变成余数 `y` 的总代价

最后答案就是所有 `x != y` 中 `evenCost[x] + oddCost[y]` 的最小值。

**代码**

```cpp
class Solution {
public:
    // calcModDistance - 计算当前余数到目标余数的最短环形距离
    int calcModDistance(int from, int to, int k) {
        int left = (from - to + k) % k;
        int right = (to - from + k) % k;
        return min(left, right);
    }

    int minOperations(vector<int>& nums, int k) {
        int n = nums.size();
        vector<int> evenCost(k, 0), oddCost(k, 0);

        for (int i = 0; i < n; i++) {
            int residue = nums[i] % k;
            for (int target = 0; target < k; target++) {
                int cost = calcModDistance(residue, target, k);
                if (i % 2 == 0) {
                    evenCost[target] += cost;
                } else {
                    oddCost[target] += cost;
                }
            }
        }

        int ans = 1000000000;
        for (int x = 0; x < k; x++) {
            for (int y = 0; y < k; y++) {
                if (x == y) continue;
                ans = min(ans, evenCost[x] + oddCost[y]);
            }
        }

        return ans;
    }
};
```

**复杂度分析**

设数组长度为 `n`。

时间复杂度：$O(nk + k^2)$。每个位置枚举所有目标余数，再枚举一次 `(x, y)` 组合。

空间复杂度：$O(k)$，只维护了两个长度为 `k` 的代价数组。

## 3938 - Maximum Path Intersection Sum in a Grid

**题意**

题目链接：[Maximum Path Intersection Sum in a Grid](https://leetcode.com/problems/maximum-path-intersection-sum-in-a-grid/description/)

给定一个 `m x n` 的整数矩阵 `grid`。

玩家 1 从左上角 `(0, 0)` 出发，只能向右或向下走，终点是右下角 `(m - 1, n - 1)`。
玩家 2 从左下角 `(m - 1, 0)` 出发，只能向右或向上走，终点是右上角 `(0, n - 1)`。

如果某个格子同时属于两条路径，那么它是共享格子。要求返回所有共享格子权值和的最大值。

**解题思路**

先刻画交集形状。设两个共享格子分别为 `(a, b)` 和 `(c, d)`，且 `b < d`。由于玩家 1 的行号只会不减，玩家 2 的行号只会不增，因此必须同时有 `a <= c` 和 `a >= c`，从而 `a == c`。这说明：不同列的共享格子只能位于同一行。

同理，不同行的共享格子只能位于同一列。

因此，两条路径的交集只能是：

- 某一行上的连续段
- 某一列上的连续段
- 一个内部单点

其中长度为 1 的交集只能发生在内部格子；边界格子无法单独作为交集，因为两条路径若在边界只共享一个格子，无法完成一次真正的横纵交叉。

于是问题就变成了三类候选的最大值：

1. 每一行中，长度至少为 `2` 的最大连续子段和；
2. 每一列中，长度至少为 `2` 的最大连续子段和；
3. 每个内部格子单独作为交集的值。

前两类可以用一个变形的 Kadane 算法处理：`current` 表示必须以当前位置结尾、且长度至少为 `2` 的最大连续子段和，转移时只需在“接在前面继续”和“从当前两个位置重新开始”之间取较大值。

**代码**

```cpp
class Solution {
public:
    // maxSubarraySumAtLeastTwo - 求长度至少为 2 的最大连续子段和
    int maxSubarraySumAtLeastTwo(vector<int>& a) {
        int n = a.size();
        int current = a[0] + a[1];
        int best = current;

        for (int i = 2; i < n; i++) {
            current = max(current + a[i], a[i - 1] + a[i]);
            best = max(best, current);
        }

        return best;
    }

    int maxScore(vector<vector<int>>& grid) {
        int m = grid.size();
        int n = grid[0].size();
        int ans = -1000000000;

        for (int i = 0; i < m; i++) {
            ans = max(ans, maxSubarraySumAtLeastTwo(grid[i]));
        }

        vector<int> column(m);
        for (int j = 0; j < n; j++) {
            for (int i = 0; i < m; i++) {
                column[i] = grid[i][j];
            }
            ans = max(ans, maxSubarraySumAtLeastTwo(column));
        }

        for (int i = 1; i + 1 < m; i++) {
            for (int j = 1; j + 1 < n; j++) {
                ans = max(ans, grid[i][j]);
            }
        }

        return ans;
    }
};
```

**复杂度分析**

设矩阵大小为 `m x n`。

每一行做一次最大子段和，列也同样扫描一次，内部单点再枚举一次，因此总时间复杂度为 $O(mn)$。

额外空间复杂度为 $O(m)$，主要是列扫描时使用了一个长度为 `m` 的临时数组。

## 3939 - Count Non Adjacent Subsets in a Rooted Tree

**题意**

题目链接：[Count Non Adjacent Subsets in a Rooted Tree](https://leetcode.com/problems/count-non-adjacent-subsets-in-a-rooted-tree/description/)

给定一棵以 `0` 为根的有根树，`parent[i]` 表示节点 `i` 的父节点。每个节点 `i` 有权值 `nums[i]`，再给定整数 `k`。统计非空节点子集的数量，满足：

- 选中节点的权值和能被 `k` 整除；
- 不能同时选择一条边的两个端点，也就是不能同时选择父子节点。

返回方案数对 `10^9 + 7` 取模后的结果。

**解题思路**

这是树形 DP。对每个节点 `u`，维护两类状态：

- `f_0[u][r]`：在 `u` 的子树中选择一些节点，且不选 `u`，总和模 `k` 为 `r` 的方案数；
- `f_1[u][r]`：在 `u` 的子树中选择一些节点，且选 `u`，总和模 `k` 为 `r` 的方案数。

初始化时：

- 不选 `u`，空集合法，所以 `f_0[u][0] = 1`
- 只选 `u`，所以 `f_1[u][nums[u] % k] = 1`

合并儿子 `v` 时分两种情况：

1. 如果当前不选 `u`，那么 `v` 可以选，也可以不选，因此儿子子树的可用方案数是 `f_0[v] + f_1[v]`。
2. 如果当前选了 `u`，那么 `v` 不能选，只能使用 `f_0[v]`。

把两个余数数组合并时，方案数相乘，余数相加取模，即新的余数是 `(i + j) % k`。

所有儿子合并完后，根节点的答案是 `f_0[0][0] + f_1[0][0]`，但其中 `f_0[0][0]` 包含空集，所以最后需要减去 `1`。

**代码**

```cpp
const int MOD = 1000000007;

class Solution {
public:
    int modBase;
    vector<vector<int>> children;
    vector<int> nodeValues;

    // mergeModuloWays - 合并两个模 k 余数 DP 数组
    vector<int> mergeModuloWays(const vector<int>& left, const vector<int>& right) {
        vector<int> merged(modBase, 0);
        for (int i = 0; i < modBase; i++) {
            if (left[i] == 0) continue;
            for (int j = 0; j < modBase; j++) {
                if (right[j] == 0) continue;
                int t = (i + j) % modBase;
                merged[t] = (merged[t] + (long long)left[i] * right[j]) % MOD;
            }
        }
        return merged;
    }

    // dfsSubtree - 返回子树的两个 DP 数组：不选当前点 / 选当前点
    pair<vector<int>, vector<int>> dfsSubtree(int u) {
        vector<int> notTake(modBase, 0), take(modBase, 0);
        notTake[0] = 1;
        take[nodeValues[u] % modBase] = 1;

        for (int v : children[u]) {
            auto [childNotTake, childTake] = dfsSubtree(v);

            vector<int> childAll(modBase, 0);
            for (int r = 0; r < modBase; r++) {
                childAll[r] = childNotTake[r] + childTake[r];
                if (childAll[r] >= MOD) {
                    childAll[r] -= MOD;
                }
            }

            notTake = mergeModuloWays(notTake, childAll);
            take = mergeModuloWays(take, childNotTake);
        }

        return {notTake, take};
    }

    int countValidSubsets(vector<int>& parent, vector<int>& nums, int k) {
        int n = parent.size();
        modBase = k;
        nodeValues = nums;
        children.assign(n, {});

        for (int i = 1; i < n; i++) {
            children[parent[i]].push_back(i);
        }

        auto [notTakeRoot, takeRoot] = dfsSubtree(0);

        int ans = notTakeRoot[0] + takeRoot[0];
        if (ans >= MOD) {
            ans -= MOD;
        }
        ans = (ans - 1 + MOD) % MOD;
        return ans;
    }
};
```

**复杂度分析**

设节点数为 `n`，模数为 `k`。

每合并一个儿子都需要做一次模意义下的卷积，复杂度为 $O(k^2)$。树上一共有 `n - 1` 条边，因此总时间复杂度为 $O(nk^2)$。

空间复杂度为 $O(nk)$，主要来自递归过程中保存的状态数组；在本题范围内可以通过。
