## 3950 - Exactly One Consecutive Set Bits Pair

**题意**

题目链接：[Exactly One Consecutive Set Bits Pair](https://leetcode.com/problems/exactly-one-consecutive-set-bits-pair/description/)

给定一个非负整数 `n`，判断它的二进制表示中是否恰好存在一对相邻的 `1`。

例如，`6` 的二进制表示为 `110`，只有一对相邻的 `1`；`7` 的二进制表示为 `111`，其中包含两对相邻的 `1`。

**解题思路**

将 `n` 右移一位后，原来相邻的两个二进制位会对齐。于是：

```cpp
int x = n & (n >> 1);
```

若 `n` 的第 `i` 位和第 `i + 1` 位都是 `1`，那么 `x` 的第 `i` 位就是 `1`。因此，`x` 中每个二进制 `1` 都与原数中的一对相邻 `1` 一一对应。

问题就转化为判断 `x` 中是否恰好只有一个 `1`，直接使用 `__builtin_popcount` 统计即可。

**代码**

```cpp
class Solution {
public:
    bool consecutiveSetBits(int n) {
        int x = n & (n >> 1);
        return __builtin_popcount(x) == 1;
    }
};
```

**复杂度分析**

在固定宽度整数模型下，时间复杂度为 $O(1)$，空间复杂度为 $O(1)$。

**变形**

如果需要统计相邻 `1` 的总对数，答案就是 `popcount(n & (n >> 1))`。如果要求判断是否存在长度至少为 `k` 的连续 `1`，可以连续执行 `k - 1` 次移位与运算：

$$
n\ \&\ (n \gg 1)\ \&\ (n \gg 2)\ \&\ \cdots\ \&\ (n \gg (k-1))
$$

结果非零就说明存在这样的连续段。

## 3951 - Minimum Energy to Maintain Brightness

**题意**

题目链接：[Minimum Energy to Maintain Brightness](https://leetcode.com/problems/minimum-energy-to-maintain-brightness/description/)

有 `n` 盏灯排成一行。打开一盏灯，可以照亮它自身以及相邻位置。题目给出若干闭区间 `intervals[i] = [l_i, r_i]`，要求在这些区间覆盖的每个整数时刻，至少照亮 `brightness` 个位置。

在某个时刻打开一盏灯会消耗一个单位的能量，求满足要求所需的最少总能量。

**解题思路**

问题可以拆成两个互相独立的部分：每个有效时刻最少需要打开多少盏灯，以及一共有多少个有效的整数时刻。

### 1. 单个时刻的最少能量

一盏灯最多照亮三个位置，所以要照亮 `brightness` 个位置，至少需要

$$
\left\lceil\frac{brightness}{3}\right\rceil
$$

盏灯。

这个下界一定可以达到：从某段连续的 `brightness` 个位置开始，每三个位置分成一组，在每组中打开中间的一盏灯；最后不足三个位置的部分再打开一盏灯即可。因此单个有效时刻的最少能量为：

```cpp
(brightness + 2) / 3
```

### 2. 统计有效整数时刻

同一个时刻即使被多个区间覆盖，也只需要满足一次亮度要求，因此需要计算所有闭区间的并集大小。

先按左端点排序，再依次合并区间。因为时间是离散的整数，若下一个区间的左端点满足 `nl <= r + 1`，两个区间可以直接合并；否则将当前区间的长度 `r - l + 1` 加入答案，并开始一个新区间。

设区间并集包含 `T` 个整数时刻，则最终答案为：

$$
T \times \left\lceil\frac{brightness}{3}\right\rceil
$$

**代码**

```cpp
using LL = long long;
class Solution {
public:
    long long minEnergy(int n, int brightness, vector<vector<int>>& intervals) {
        sort(intervals.begin(), intervals.end());

        LL sum = 0;
        LL l = intervals[0][0];
        LL r = intervals[0][1];

        for (int i = 1; i < (int)intervals.size(); i++) {
            LL nl = intervals[i][0];
            LL nr = intervals[i][1];

            if (nl <= r + 1) {
                r = max(r, nr);
            } else {
                sum += r - l + 1;
                l = nl;
                r = nr;
            }
        }

        sum += r - l + 1;

        LL cnt = (brightness + 2LL) / 3;
        return sum * cnt;
    }
};
```

**复杂度分析**

设区间数量为 $m$。

时间复杂度：$O(m\log m)$，主要来自区间排序；合并过程为 $O(m)$。

空间复杂度：$O(\log m)$，来自排序所需的递归栈；除排序外只使用了常数个额外变量。

**变形**

如果时间改为连续实数，区间长度应改成 `r - l`，相邻但不重叠的区间也不再需要用 `r + 1` 合并。如果每个时刻的亮度要求不同，可以先用差分事件把时间轴切成若干个亮度恒定的区间，再分别计算每段的能量。

## 3952 - Maximum Total Value of Covered Indices

**题意**

题目链接：[Maximum Total Value of Covered Indices](https://leetcode.com/problems/maximum-total-value-of-covered-indices/description/)

给定正整数数组 `nums` 和一个等长的二进制字符串 `s`。`s[i] = '1'` 表示位置 `i` 初始有一个标记。

每个标记最多移动一次：位于 `i > 0` 的标记可以留在原位置，也可以向左移动到 `i - 1`。最终至少有一个标记的位置称为被覆盖位置，目标是最大化所有被覆盖位置的 `nums[i]` 之和。

**解题思路**

将 `s` 分解成若干个极大的连续 `1` 段，考虑其中一段 `[l, r]`。这一段有

$$
k = r-l+1
$$

个标记。

### 1. `l > 0` 时的候选位置

每个标记只能留在原地或向左移动一格，因此这一段标记能够覆盖的位置只可能位于 `[l - 1, r]`。这个区间一共有 `k + 1` 个位置，但只有 `k` 个标记，所以最多只能覆盖其中 `k` 个位置，也就是必须舍弃一个位置。

并且 `[l - 1, r]` 中任意一个位置 `p` 都可以被舍弃：

- 让 `[l, p]` 中的标记全部向左移动一格；
- 让 `[p + 1, r]` 中的标记全部留在原地。

这样正好覆盖 `[l - 1, r]` 中除 `p` 以外的所有位置。因此，为了让权值和最大，只需舍弃其中权值最小的位置。这一段的贡献为：

$$
\sum_{i=l-1}^{r} nums[i] - \min_{i=l-1}^{r} nums[i]
$$

### 2. `l = 0` 时的边界情况

最左侧的标记无法再向左移动。此时候选区间就是 `[0, r]`，位置数量与标记数量相同，所有位置都可以被覆盖，这一段直接贡献 `nums[0..r]` 的总和。

不同连续 `1` 段之间至少隔着一个 `0`，它们对应的候选区间互不相交，因此每一段可以独立取最优解，最后累加即可。

**代码**

```cpp
using LL = long long;
using ULL = unsigned long long;

class Solution {
public:
    long long maxTotal(vector<int>& nums, string s) {
        int n = nums.size();
        LL ans = 0;

        for (int i = 0; i < n; ) {
            if (s[i] == '0') {
                i++;
                continue;
            }

            int l = i;
            LL sm = 0;
            int mn = INT_MAX;

            while (i < n && s[i] == '1') {
                sm += nums[i];
                mn = min(mn, nums[i]);
                i++;
            }

            if (l == 0) {
                ans += sm;
            } else {
                sm += nums[l - 1];
                mn = min(mn, nums[l - 1]);
                ans += sm - mn;
            }
        }

        return ans;
    }
};
```

**复杂度分析**

时间复杂度：$O(n)$，每个位置最多被扫描一次。

空间复杂度：$O(1)$，只使用了常数个额外变量。

**变形**

这类问题的关键是把“每个标记可以向左移动一格”转化成连续段的可覆盖区间。如果每个标记最多可以向左移动 `k` 格，连续段之间可能发生相互影响，候选区间也会重叠，通常需要使用动态规划或匹配方法处理，而不能再对每个连续段独立贪心。

## 3953 - Maximum Score with Co-Prime Element

**题意**

题目链接：[Maximum Score with Co-Prime Element](https://leetcode.com/problems/maximum-score-with-co-prime-element/description/)

给定正整数数组 `nums` 和整数 `maxVal`。可以进行若干次修改，每次把一个数组元素改成不超过 `maxVal` 的正整数，代价为 `1`。

修改完成后，需要选择一个下标，使该位置的值 `x` 与其他所有元素都互质。最终得分为 `x` 减去修改次数，求能够得到的最大得分。

**解题思路**

设最终选中的值为 `x`，定义：

$$
bad(x)=|\{a\in nums\mid \gcd(a,x)>1\}|
$$

也就是原数组中与 `x` 不互质的元素数量。只要能够快速求出每个候选 `x` 的 `bad(x)`，就可以直接计算答案。

### 1. `x` 已经存在于原数组

如果 `x > 1`，被选中的那个 `x` 也会被计入 `bad(x)`，但它作为选中元素不需要修改。只需把其余 `bad(x) - 1` 个冲突元素改成 `1`，最小修改次数为 `bad(x) - 1`。

当 `x = 1` 时，它天然与所有元素互质，修改次数为 `0`。两种情况可以统一写成：

$$
score(x)=x-bad(x)+[x>1]
$$

其中 `[x > 1]` 表示条件成立时取 `1`，否则取 `0`。

### 2. `x` 不存在于原数组

此时必须先把某个元素修改成 `x`，所以只有 `x <= maxVal` 才是合法候选。

- 若 `bad(x) > 0`，可以把一个冲突元素改成 `x` 并选中它，再把剩余的 `bad(x) - 1` 个冲突元素改成 `1`，总修改次数为 `bad(x)`；
- 若 `bad(x) = 0`，仍然需要修改一个元素来创造 `x`，总修改次数为 `1`。

因此：

$$
score(x)=x-\max(1,bad(x))
$$

注意，原数组中已经存在的值即使大于 `maxVal`，仍然可以保持不变并成为选中元素。设

$$
V=\max(maxVal,\max(nums))
$$

枚举 `1..V` 就覆盖了所有可能的候选值。

### 3. 用 Möbius 反演计算 `bad(x)`

直接对每个候选 `x` 遍历数组并计算最大公约数，需要 $O(nV\log V)$ 的时间。这里需要利用 Möbius 反演，把“与 `x` 互质”转化成若干个“同时被 `d` 整除”的计数。

#### 3.1 Möbius 函数及其基本恒等式

Möbius 函数 $μ(m)$ 的定义为：

$$
\mu(m)=
\begin{cases}
1, & m=1 \\
0, & m\text{ 含有平方质因子} \\
(-1)^k, & m\text{ 是 }k\text{ 个不同质数的乘积}
\end{cases}
$$

它满足如下恒等式：

$$
\sum_{d\mid m}\mu(d)=
\begin{cases}
1, & m=1 \\
0, & m>1
\end{cases}
$$

右侧也可以写成指示函数 $[m=1]$。令 $m=\gcd(a,x)$，即可得到：

$$
[\gcd(a,x)=1]
=\sum_{d\mid \gcd(a,x)}\mu(d)
$$

原因是：当 `a` 与 `x` 互质时，最大公约数为 `1`，右侧只有 $μ(1)=1$；当最大公约数大于 `1` 时，它的所有因数对应的 Möbius 函数之和为 `0`。

#### 3.2 统计每个因数的倍数个数

先令 `cnt[v]` 表示数值 `v` 在原数组中的出现次数。对每个 `d` 定义：

$$
mul[d]=\sum_{k=1}^{\lfloor V/d\rfloor}cnt[kd]
$$

`mul[d]` 表示原数组中能被 `d` 整除的元素数量。代码中固定 `d`，依次枚举 `d, 2d, 3d, ...`，即可累加出这个值：

```cpp
for (int d = 1; d <= v; d++) {
    for (int j = d; j <= v; j += d) {
        mul[d] += cnt[j];
    }
}
```

#### 3.3 推导 `cop(x)`

令 `cop(x)` 表示原数组中与 `x` 互质的元素数量。把每个元素是否与 `x` 互质的指示函数相加：

$$
\begin{aligned}
cop(x)
&=\sum_{a\in nums}[\gcd(a,x)=1] \\
&=\sum_{a\in nums}\sum_{d\mid\gcd(a,x)}\mu(d) \\
&=\sum_{a\in nums}\sum_{\substack{d\mid x\\d\mid a}}\mu(d) \\
&=\sum_{d\mid x}\mu(d)\sum_{\substack{a\in nums\\d\mid a}}1 \\
&=\sum_{d\mid x}\mu(d)\cdot mul[d]
\end{aligned}
$$

第三行只是把“`d` 是 `gcd(a, x)` 的因数”拆成了“`d | a` 且 `d | x`”。第四行交换了枚举顺序：先枚举 `x` 的因数 `d`，再统计数组中有多少元素能被 `d` 整除；后者正是预处理得到的 `mul[d]`。

得到 `cop(x)` 后，与 `x` 不互质的元素数量就是：

$$
bad(x)=n-cop(x)
$$

例如 `nums = [2, 3, 4, 9]`，考虑 `x = 10`。`10` 的因数为 `1, 2, 5, 10`，对应的 Möbius 函数值为 `1, -1, -1, 1`，并且：

$$
mul[1]=4,\quad mul[2]=2,\quad mul[5]=0,\quad mul[10]=0
$$

所以：

$$
cop(10)=4-2-0+0=2
$$

这两个互质元素正是 `3` 和 `9`，因此 `bad(10) = 4 - 2 = 2`。

#### 3.4 批量计算所有 `cop[x]`

公式要求对每个 `x` 枚举它的所有因数。实现时反过来枚举：固定因数 `d`，将它对答案的贡献

$$
\mu(d)\cdot mul[d]
$$

加到 `d` 的所有倍数 `x` 上：

```cpp
for (int d = 1; d <= v; d++) {
    if (mu[d] == 0) {
        continue;
    }

    int val = mu[d] * mul[d];
    for (int x = d; x <= v; x += d) {
        cop[x] += val;
    }
}
```

因为此处遍历的 `x` 都满足 `d | x`，所以每个 `cop[x]` 最终恰好收到了它所有因数 `d` 的贡献。

代码使用线性筛计算 Möbius 函数：

- `mu[1] = 1`；
- 若 `i` 是新发现的质数，则 `mu[i] = -1`；
- 枚举质数 `p` 时，若 `p | i`，则 `i * p` 含有平方质因子 $p^2$，因此 `mu[i * p] = 0`，并立即结束当前枚举；
- 若 `p` 不能整除 `i`，则 `i * p` 比 `i` 多一个不同的质因子，因此 `mu[i * p] = -mu[i]`。

预处理 `mul[d]` 和累加 `cop[x]` 都是枚举每个整数的倍数，总枚举次数为 $O(V\log V)$。

**代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

#ifdef LOCAL
#include "debug.h"
#else
#define debug(...) 42
#endif
using LL = long long;
using ULL = unsigned long long;

class Solution {
public:
    int maxScore(vector<int>& nums, int maxVal) {
        int n = nums.size();
        int v = maxVal;

        for (int x : nums) {
            v = max(v, x);
        }

        vector<int> cnt(v + 1);
        for (int x : nums) {
            cnt[x]++;
        }

        vector<int> mu(v + 1);
        vector<int> ps;
        vector<bool> vis(v + 1);

        mu[1] = 1;

        for (int i = 2; i <= v; i++) {
            if (!vis[i]) {
                ps.push_back(i);
                mu[i] = -1;
            }

            for (int p : ps) {
                if (p > v / i) {
                    break;
                }

                int x = i * p;
                vis[x] = true;

                if (i % p == 0) {
                    mu[x] = 0;
                    break;
                }

                mu[x] = -mu[i];
            }
        }

        vector<int> mul(v + 1);

        for (int d = 1; d <= v; d++) {
            for (int j = d; j <= v; j += d) {
                mul[d] += cnt[j];
            }
        }

        vector<int> cop(v + 1);

        for (int d = 1; d <= v; d++) {
            if (mu[d] == 0) {
                continue;
            }

            int val = mu[d] * mul[d];

            for (int x = d; x <= v; x += d) {
                cop[x] += val;
            }
        }

        int ans = 0;

        for (int x = 1; x <= v; x++) {
            int bad = n - cop[x];

            if (cnt[x] > 0) {
                int cur = x - bad + (x > 1);
                ans = max(ans, cur);
            } else if (x <= maxVal) {
                int cur = x - max(1, bad);
                ans = max(ans, cur);
            }
        }

        return ans;
    }
};
```

**复杂度分析**

设 $V=\max(maxVal,\max(nums))$。

线性筛计算 Möbius 函数需要 $O(V)$。两次倍数枚举的总次数均为：

$$
\sum_{d=1}^{V}\left\lfloor\frac{V}{d}\right\rfloor=O(V\log V)
$$

因此总时间复杂度为 $O(n+V\log V)$，空间复杂度为 $O(V)$。

**变形**

当问题需要对大量候选 `x` 统计数组中满足 `gcd(a, x) = 1` 的元素数量时，Möbius 反演通常比逐个计算最大公约数更高效。若只查询少量候选值，则可以分解每个 `x` 的不同质因子，再用容斥统计与它不互质的元素，避免预处理整个值域。
