## 3963 - Create Grid With Exactly One Path

**题意**

题目链接：[Create Grid With Exactly One Path](https://leetcode.com/problems/create-grid-with-exactly-one-path/description/)

给定网格的行数 `m` 和列数 `n`，需要构造一个只包含 `'.'` 和 `'#'` 的 `m \times n` 网格：

- `'.'` 表示可以经过的空地；
- `'#'` 表示不能经过的障碍。

路径从左上角 `(0,0)` 出发，只能向右或向下移动，最终到达右下角 `(m-1,n-1)`。要求构造出的网格中恰好只有一条合法路径。

- `1 <= m, n <= 25`

**解题思路**

题目允许返回任意合法构造，因此只需要预先选定一条简单路径，再把其他位置全部设为障碍。

最直接的选择是：

1. 沿第一行一直向右，从 `(0,0)` 走到 `(0,n-1)`；
2. 沿最后一列一直向下，走到 `(m-1,n-1)`。

因此，先把整个网格初始化为 `'#'`，再将第一行和最后一列全部改为 `'.'`。

这条路径一定存在。下面说明它为什么是唯一的：

- 在第一行中，除了继续向右以外，向下的位置都是障碍；只有到达最后一列后才能向下；
- 在最后一列中已经不能继续向右，只能不断向下直到终点；
- 其余位置全部是障碍，不可能形成分支或绕行路径。

当 `m=1` 或 `n=1` 时，构造会退化成唯一的一行或一列，结论仍然成立。

**代码**

```cpp
class Solution {
public:
    vector<string> createGrid(int m, int n) {
        vector<string> g(m, string(n, '#'));

        for (int j = 0; j < n; j++) {
            g[0][j] = '.';
        }

        for (int i = 0; i < m; i++) {
            g[i][n - 1] = '.';
        }

        return g;
    }
};
```

**复杂度分析**

构造并返回整个网格需要访问 `m \times n` 个位置。

时间复杂度：`O(mn)`。

空间复杂度：`O(mn)`，用于保存返回的网格；除返回值外的额外空间复杂度为 `O(1)`。

**变形**

如果要求路径经过某个指定位置 `(x,y)`，可以先向右走到第 `y` 列，再向下走到第 `x` 行，然后继续选取一条不产生分支的折线路径到达终点。

如果要求构造恰好 `k` 条路径，就不能只保留一条折线，需要利用局部矩形或分层结构控制路径数量。由于只允许向右和向下移动，某个空地的到达路径数满足：

$$
dp[i][j]=dp[i-1][j]+dp[i][j-1]
$$

这时可以把构造问题转化为控制网格上的路径计数。

## 3964 - Minimum Lights to Illuminate a Road

**题意**

题目链接：[Minimum Lights to Illuminate a Road](https://leetcode.com/problems/minimum-lights-to-illuminate-a-road/description/)

给定长度为 `n` 的数组 `lights`，表示道路上的 `n` 个位置。

若 `lights[i]=v>0`，位置 `i` 有一盏正常工作的灯，可以照亮闭区间：

$$
[\max(0,i-v),\min(n-1,i+v)]
$$

若 `lights[i]=0`，则该位置没有正常工作的灯。

可以在任意位置安装额外的灯，每盏额外的灯可以照亮自身及相邻位置，即半径为 `1`。求使整条道路都被照亮所需的最少额外灯数。

- `1 <= n <= 10^5`
- `0 <= lights[i] <= n`

**解题思路**

问题分成两个阶段：

1. 用差分数组标记已经被原有灯照亮的位置；
2. 对剩余位置从左到右贪心安装额外的灯。

### 1. 差分数组标记已有覆盖

每盏正常工作的灯对应一个覆盖区间 `[l,r]`。如果直接枚举区间内的每个位置，最坏时间复杂度会达到 `O(n^2)`。

使用差分数组 `dif`，对于每个覆盖区间执行：

```cpp
dif[l]++;
dif[r + 1]--;
```

随后计算前缀和。当前位置的前缀和大于 `0`，就说明它至少被一盏原有灯照亮。

### 2. 覆盖最左侧的不可见位置

从左到右扫描道路。设 `i` 是当前最左侧尚未被照亮的位置，那么新安装的灯必须覆盖 `i`。

额外的灯覆盖半径为 `1`。在所有能够覆盖 `i` 的安装位置中，应当把灯尽量放在右边，也就是 `i+1`；若 `i` 已经是最后一个位置，则放在边界位置即可。这样可以在覆盖 `i` 的同时，把右侧覆盖范围扩展到最远。

一次安装最多覆盖：

$$
i,\ i+1,\ i+2
$$

因此安装一盏灯后，可以直接令 `i += 3`。如果当前位置原本已经可见，则只需令 `i++`。

### 3. 贪心正确性

考虑最左侧不可见位置 `i`。任何可行方案都必须安装一盏能够覆盖 `i` 的新灯。

由于 `i` 左侧的所有位置都已经可见，把这盏灯向右移动到仍能覆盖 `i` 的最右位置，不会破坏左侧的覆盖情况，却能覆盖不少于原方案的右侧位置。因此一定存在一个最优方案，其第一盏新增灯采用当前的贪心选择。

覆盖完这段后，剩余问题与原问题具有相同结构，可以继续使用同样的选择。由此得到全局最优解。

**代码**

```cpp
using LL = long long;
using ULL = unsigned long long;

class Solution {
public:
    int minLights(vector<int>& lights) {
        int n = lights.size();
        vector<int> dif(n + 1);

        for (int i = 0; i < n; i++) {
            int v = lights[i];

            if (v == 0) {
                continue;
            }

            int l = max(0, i - v);
            int r = min(n - 1, i + v);

            dif[l]++;
            dif[r + 1]--;
        }

        vector<int> vis(n);
        int sum = 0;

        for (int i = 0; i < n; i++) {
            sum += dif[i];
            vis[i] = sum > 0;
        }

        int ans = 0;

        for (int i = 0; i < n;) {
            if (vis[i]) {
                i++;
            } else {
                ans++;
                i += 3;
            }
        }

        return ans;
    }
};
```

**复杂度分析**

每盏原有灯只进行一次差分更新，之后分别进行一次前缀和与贪心扫描。

时间复杂度：`O(n)`。

空间复杂度：`O(n)`。

**变形**

如果额外安装的灯的照明半径改为 `p`，遇到最左不可见位置 `i` 时，应把灯安装在不越界的最右位置 `i+p`，一次最多覆盖从 `i` 开始的 `2p+1` 个位置。

如果每个位置安装灯的代价不同，最右放置不再一定最优。这时需要在能覆盖当前位置的候选灯中选择代价最小的灯，可以使用动态规划或优先队列维护滑动窗口内的最优选择。

## 3965 - Finish Time of Tasks I

**题意**

题目链接：[Finish Time of Tasks I](https://leetcode.com/problems/finish-time-of-tasks-i/description/)

有 `n` 个任务组成一棵以任务 `0` 为根的有根树。`edges[i]=[u_i,v_i]` 表示 `u_i` 是 `v_i` 的父任务，`baseTime[i]` 表示任务 `i` 的基础执行时间。

每个任务的完成时间按下面的规则计算：

- 叶子任务的完成时间为 `baseTime[i]`；
- 对于非叶子任务，设所有孩子的完成时间最小值为 `earliest`，最大值为 `latest`；
- 当前任务自身的执行时间为 `(latest-earliest)+baseTime[i]`；
- 当前任务的完成时间为 `latest+ownDuration`。

求根任务 `0` 的完成时间。

- `1 <= n <= 10^5`
- `edges.length = n - 1`
- `1 <= baseTime[i] <= 10^5`

**解题思路**

父任务的完成时间依赖所有孩子的完成时间，因此需要按照后序顺序计算树形 DP。

定义：

$$
dp[u]=\text{任务 }u\text{ 的完成时间}
$$

### 1. 化简状态转移

如果 `u` 是叶子：

$$
dp[u]=baseTime[u]
$$

如果 `u` 不是叶子，设：

$$
mn=\min_{v\in child(u)}dp[v]
$$

$$
mx=\max_{v\in child(u)}dp[v]
$$

根据题意：

$$
ownDuration=(mx-mn)+baseTime[u]
$$

所以：

$$
\begin{aligned}
dp[u]
&=mx+ownDuration\\
&=mx+(mx-mn)+baseTime[u]\\
&=2mx-mn+baseTime[u]
\end{aligned}
$$

因此，每个节点只需要维护所有孩子完成时间的最小值和最大值。

### 2. 迭代生成后序

`n` 最大为 `10^5`，递归 DFS 可能因为树退化成链而导致调用栈过深。代码使用迭代方式：

1. 从根节点 `0` 出发，按照父到子的方向生成遍历序列 `ord`；
2. 逆序遍历 `ord`。

由于父节点一定先于孩子进入 `ord`，逆序后孩子就一定先于父节点被计算，等价于后序遍历。

**代码**

```cpp
using LL = long long;
using ULL = unsigned long long;

class Solution {
public:
    long long finishTime(int n, vector<vector<int>>& edges,
                         vector<int>& baseTime) {
        vector<vector<int>> g(n);

        for (auto& e : edges) {
            int u = e[0];
            int v = e[1];
            g[u].push_back(v);
        }

        vector<int> ord;
        ord.reserve(n);
        ord.push_back(0);

        for (int i = 0; i < (int)ord.size(); i++) {
            int u = ord[i];

            for (int v : g[u]) {
                ord.push_back(v);
            }
        }

        vector<LL> dp(n);

        for (int i = n - 1; i >= 0; i--) {
            int u = ord[i];

            if (g[u].empty()) {
                dp[u] = baseTime[u];
                continue;
            }

            LL mn = LLONG_MAX;
            LL mx = LLONG_MIN;

            for (int v : g[u]) {
                mn = min(mn, dp[v]);
                mx = max(mx, dp[v]);
            }

            dp[u] = 2 * mx - mn + baseTime[u];
        }

        return dp[0];
    }
};
```

**复杂度分析**

每个节点和每条边都只会被处理常数次。

时间复杂度：`O(n)`。

空间复杂度：`O(n)`，用于邻接表、遍历序列和 DP 数组。

**变形**

如果输入的边没有指定父子方向，需要从根节点出发记录每个节点的父亲，避免沿无向边返回父节点。

如果转移依赖孩子完成时间的中位数、第 `k` 小值等信息，仅维护最小值和最大值就不够了，可以在每个节点处使用排序、堆或顺序统计数据结构。

## 3966 - Count Good Integers in a Range

**题意**

题目链接：[Count Good Integers in a Range](https://leetcode.com/problems/count-good-integers-in-a-range/description/)

给定整数 `l`、`r` 和 `k`。如果一个整数的十进制表示中，每一对相邻数位的绝对差都不超过 `k`，则称它为好整数。

求闭区间 `[l,r]` 中好整数的数量。

- `10 <= l <= r <= 10^{15}`
- `0 <= k <= 9`

**解题思路**

区间上界达到 `10^{15}`，无法直接枚举。先定义：

$$
F(x)=\text{区间 }[1,x]\text{ 中好整数的数量}
$$

那么原问题的答案为：

$$
F(r)-F(l-1)
$$

接下来使用数位 DP 计算 `F(x)`。

### 1. DP 状态

将 `x` 转换成十进制数位数组 `dig`，从高位到低位填写数字。

`dfs(pos,pre,lim)` 表示从第 `pos` 位开始填写时的合法方案数：

- `pos`：当前处理到的数位下标；
- `pre`：前一个有效数位，取值为 `0` 到 `9`；
- `pre=10`：尚未填写第一个非零数位，即当前仍处于前导零阶段；
- `lim`：当前前缀是否与 `x` 的前缀完全相同。

若 `lim=true`，当前位最多只能填写 `dig[pos]`；否则可以填写 `0` 到 `9`。

### 2. 前导零的处理

前导零不属于整数的十进制表示，不能参与相邻数位差的判断。

当 `pre=10` 且当前填写 `0` 时，下一层仍保持 `pre=10`。一旦填写了第一个非零数字 `d`，就令 `pre=d`，之后每个新数位都必须满足：

$$
|pre-d|\le k
$$

到达数位末尾时，只有 `pre\ne 10` 才计入答案。这样会排除所有数位均为前导零的方案，也就是数字 `0`。

例如统计不超过 `305` 的好整数时，数字 `27` 会以 `027` 的形式参与 DP：

- 第一位 `0` 是前导零，不参与相邻差判断；
- `2` 是第一个有效数位；
- 只需要检查 `|2-7|\le k`。

### 3. 上界约束

若当前状态满足 `lim=true`，说明此前填写的前缀与 `x` 完全相同。设当前位填写 `d`，新的上界状态为：

```cpp
bool nlim = lim && d == up;
```

当 `lim=true` 时，`up=dig[pos]`，只有填写上界数位才能继续保持限制；填写更小的数字后，后续各位都可以自由选择。

### 4. 记忆化

当 `lim=false` 时，后续方案数只由 `pos` 和 `pre` 决定，可以使用：

```cpp
mem[pos][pre]
```

进行记忆化。

带上界限制的状态不能直接复用，因为它还取决于 `x` 的对应后缀。每次计算新的 `F(x)` 时，也需要重新初始化记忆化数组。

**代码**

```cpp
using LL = long long;
using ULL = unsigned long long;

class Solution {
    vector<int> dig;
    vector<vector<LL>> mem;
    int kk;

    // 统计从第 pos 位开始填写时的合法数字数量
    LL dfs(int pos, int pre, bool lim) {
        if (pos == (int)dig.size()) {
            return pre != 10;
        }

        if (!lim && mem[pos][pre] != -1) {
            return mem[pos][pre];
        }

        int up = lim ? dig[pos] : 9;
        LL ans = 0;

        for (int d = 0; d <= up; d++) {
            bool nlim = lim && d == up;

            if (pre == 10 && d == 0) {
                ans += dfs(pos + 1, 10, nlim);
            } else if (pre == 10) {
                ans += dfs(pos + 1, d, nlim);
            } else if (abs(pre - d) <= kk) {
                ans += dfs(pos + 1, d, nlim);
            }
        }

        if (!lim) {
            mem[pos][pre] = ans;
        }

        return ans;
    }

    // 统计区间 [1, x] 中好整数的数量
    LL cnt(LL x) {
        if (x <= 0) {
            return 0;
        }

        dig.clear();

        string s = to_string(x);
        for (char c : s) {
            dig.push_back(c - '0');
        }

        mem.assign(dig.size(), vector<LL>(11, -1));
        return dfs(0, 10, true);
    }

public:
    long long goodIntegers(long long l, long long r, int k) {
        kk = k;
        return cnt(r) - cnt(l - 1);
    }
};
```

**复杂度分析**

设上界 `x` 的十进制位数为 `D`。非上界状态共有 `D \times 11` 个，每个状态最多枚举 `10` 个数位。

单次计算 `F(x)` 的时间复杂度：`O(D \times 11 \times 10)`，通常简写为 `O(D)`。

空间复杂度：`O(D \times 11)`，递归栈深度为 `O(D)`。

本题中 `D\le 16`，状态数量很小。

**变形**

如果要求相邻数位的绝对差恰好等于 `k`，只需把转移条件改为：

```cpp
abs(pre - d) == k
```

如果要求相邻两个数位不能相同，则禁止 `pre == d`；如果要求统计满足条件的数字之和，可以让 DP 同时返回“方案数”和“数字总和”，在加入新数位时按位权更新总和。

当限制不只依赖前一个数位，而是依赖最近若干位或某个禁止子串时，可以把 `pre` 扩展为自动机状态，再进行相同形式的数位 DP。
