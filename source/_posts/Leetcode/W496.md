## 3889 - Mirror Frequency Distance

**题意**

题目链接：[3889. Mirror Frequency Distance](https://leetcode.com/problems/mirror-frequency-distance/)

给定只含小写字母和数字的字符串。定义字母按字母表首尾镜像（如 `a` 对 `z`），数字按 `0` 到 `9` 首尾镜像（如 `0` 对 `9`）。对每个镜像对只计算一次贡献，贡献为两字符出现次数之差的绝对值，求总和。

**解题思路**

字符全集只有 $26+10=36$ 种，直接计数即可。

1. 先统计每个字符频次。
2. 枚举 13 组字母镜像对：`a-z` 到 `m-n`。
3. 枚举 5 组数字镜像对：`0-9` 到 `4-5`。
4. 累加每组的 $|freq(c)-freq(m)|$。

因为只枚举前半段，每个镜像对恰好统计一次。

**代码**

```cpp
class Solution {
public:
    int mirrorFrequency(string s) {
        vector<int> cnt(128, 0);
        for (char c : s) cnt[c]++;

        int ans = 0;
        for (int i = 0; i < 13; ++i) {
            char a = char('a' + i);
            char b = char('z' - i);
            ans += abs(cnt[a] - cnt[b]);
        }
        for (int i = 0; i < 5; ++i) {
            char a = char('0' + i);
            char b = char('9' - i);
            ans += abs(cnt[a] - cnt[b]);
        }
        return ans;
    }
};
```

**复杂度分析**

时间复杂度：$O(n)$。空间复杂度：$O(1)$。

**变形**

可扩展为「动态修改字符并多次询问」：维护全局频次与答案增量，每次修改做到均摊 $O(1)$。

## 3890 - Integers With Multiple Sum of Two Cubes

**题意**

题目链接：[3890. Integers With Multiple Sum of Two Cubes](https://leetcode.com/problems/integers-with-multiple-sum-of-two-cubes/)

给定 $n$，找出所有 $x\le n$，满足存在至少两组不同正整数对 $(a,b)$（且 $a\le b$）使得 $x=a^3+b^3$。

**解题思路**

由 $a^3\le n\le 10^9$ 可知 $a,b\le 1000$，枚举规模很小。

1. 预处理 $1^3$ 到 $1000^3$。
2. 枚举 $a=1..1000$，$b=a..1000$，计算 $s=a^3+b^3$。
3. 若 $s>n$，由于 $b$ 单调增，当前内层可直接 `break`。
4. 用哈希表统计每个和出现次数。
5. 取出现次数至少为 2 的和并排序。

限制 $a\le b$ 后，天然去掉了顺序重复对。

**代码**

```cpp
class Solution {
public:
    vector<int> findGoodIntegers(int n) {
        const int LIM = 1000;
        vector<long long> cube(LIM + 1, 0);
        for (int i = 1; i <= LIM; ++i) cube[i] = 1LL * i * i * i;

        unordered_map<int, int> cnt;
        for (int a = 1; a <= LIM; ++a) {
            if (cube[a] + cube[a] > n) break;
            for (int b = a; b <= LIM; ++b) {
                long long s = cube[a] + cube[b];
                if (s > n) break;
                cnt[(int)s]++;
            }
        }

        vector<int> ans;
        for (auto &it : cnt) {
            if (it.second >= 2) ans.push_back(it.first);
        }
        sort(ans.begin(), ans.end());
        return ans;
    }
};
```

**复杂度分析**

记 $m=\lfloor\sqrt[3]{n}\rfloor$，时间复杂度约为 $O(m^2+k\log k)$，其中 $k$ 是答案个数；在本题范围内 $m\le 1000$。空间复杂度：$O(m^2)$（哈希统计的最坏不同和数量）。

## 3891 - Minimum Increase to Maximize Special Indices

**题意**

题目链接：[3891. Minimum Increase to Maximize Special Indices](https://leetcode.com/problems/minimum-increase-to-maximize-special-indices/)

给定数组 `nums`，可对任意位置执行 `+1`。下标 $i$（$0<i<n-1$）是 special 当且仅当 $nums_i>nums_{i-1}$ 且 $nums_i>nums_{i+1}$。目标是先让 special 数量最大，再在该前提下最小化总操作次数。

**解题思路**

先把每个位置单独成为 special 的最小代价算出来：

$$
cost_i = \max\bigl(0,\max(nums_{i-1}, nums_{i+1}) + 1 - nums_i\bigr),\quad 1\le i\le n-2.
$$

关键点：

1. special 位置不能相邻，因此在内部下标区间上是“不相邻选点”。
2. 若选定某点为 special，最优只增该点本身，不会去改邻居。
3. 原题转化为：在一条链上选若干不相邻点，先最大化选点数，再最小化总代价。

用滚动 DP，维护两类状态：

- $f0$：处理到当前位置时，当前位置不选的最优二元组 $(cnt, cost)$。
- $f1$：处理到当前位置时，当前位置已选的最优二元组 $(cnt, cost)$。

比较规则：先比 $cnt$ 大小，再比 $cost$ 更小。

转移：

- 不选当前点：$g0=\max(f0,f1)$（按上述比较规则）。
- 选当前点：$g1=f0+(1,w)$，其中 $w=cost_i$。

最终答案是 $\max(f0,f1)$ 的第二维。

**代码**

```cpp
class Solution {
public:
    struct State {
        int cnt;
        long long cost;
    };

    static bool better(const State& a, const State& b) {
        if (a.cnt != b.cnt) return a.cnt > b.cnt;
        return a.cost < b.cost;
    }

    long long minIncrease(vector<int>& nums) {
        const long long INF = (1LL << 62);

        State f0{0, 0};
        State f1{-1000000000, INF};

        int n = (int)nums.size();
        for (int i = 1; i <= n - 2; ++i) {
            long long w = max(0LL, max(1LL * nums[i - 1], 1LL * nums[i + 1]) + 1 - 1LL * nums[i]);

            State g0 = better(f0, f1) ? f0 : f1;
            State g1{f0.cnt + 1, f0.cost + w};

            f0 = g0;
            f1 = g1;
        }

        State ans = better(f0, f1) ? f0 : f1;
        return ans.cost;
    }
};
```

**复杂度分析**

时间复杂度：$O(n)$。空间复杂度：$O(1)$。

**变形**

若题目改为“至少要有 $t$ 个 special 的最小代价”，可改成显式计数维的 DP：$dp_{i,j,sel}$。

## 3892 - Minimum Operations to Achieve At Least K Peaks

**题意**

题目链接：[3892. Minimum Operations to Achieve At Least K Peaks](https://leetcode.com/problems/minimum-operations-to-achieve-at-least-k-peaks/)

给定长度为 $n$ 的环形数组，可执行任意次 `+1` 操作。一个位置是峰值当且仅当它严格大于左右邻居（环形邻接）。求最少操作次数使峰值数量至少为 $k$；若无解返回 $-1$。

**解题思路**

先做三个等价转化。

1. 任何两个相邻位置不能同时是峰值，因此可选峰值集合是环上的独立集，最大峰值数为 $\lfloor n/2\rfloor$。若 $k>\lfloor n/2\rfloor$ 直接无解。
2. 对每个位置预处理“单点成为峰值”的最小代价：

$$
w_i = \max\bigl(0,\max(nums_{i-1}, nums_{i+1}) + 1 - nums_i\bigr).
$$

3. 因为所有代价 $w_i\ge 0$，“至少 $k$ 个峰值”的最优解等价于“恰好 $k$ 个峰值”的最优解。

于是问题变成：在环上选恰好 $k$ 个不相邻点，使 $\sum w_i$ 最小。

环转链分两类：

1. 不选位置 0：在线性区间 $[1,n-1]$ 里选 $k$ 个不相邻点。
2. 选位置 0：总代价先加 $w_0$，然后在线性区间 $[2,n-2]$ 里选 $k-1$ 个不相邻点。

两种情况取最小。

链上子问题 DP（恰选）如下：

- $f0_j$：处理到当前位置时，前一个位置不选，且已选 $j$ 个的最小代价。
- $f1_j$：处理到当前位置时，前一个位置已选，且已选 $j$ 个的最小代价。

转移：

$$
\begin{aligned}
g0_j &= \min(f0_j, f1_j),\\
g1_j &= f0_{j-1} + w_i.
\end{aligned}
$$

滚动更新即可，单次链 DP 为 $O(m\cdot need)$。

**代码**

```cpp
class Solution {
public:
    static constexpr long long INF = (1LL << 60);

    // 在线性区间 [l, r] 上，选恰好 need 个不相邻位置的最小代价
    long long solveLine(const vector<long long>& w, int l, int r, int need) {
        if (need < 0) return INF;
        if (need == 0) return 0;

        int m = (l <= r ? r - l + 1 : 0);
        if (m == 0) return INF;
        if (need > (m + 1) / 2) return INF;

        vector<long long> f0(need + 1, INF), f1(need + 1, INF);
        vector<long long> g0(need + 1, INF), g1(need + 1, INF);
        f0[0] = 0;

        for (int i = l; i <= r; ++i) {
            fill(g0.begin(), g0.end(), INF);
            fill(g1.begin(), g1.end(), INF);

            for (int j = 0; j <= need; ++j) {
                g0[j] = min(f0[j], f1[j]);
                if (j > 0 && f0[j - 1] < INF) {
                    g1[j] = f0[j - 1] + w[i];
                }
            }
            swap(f0, g0);
            swap(f1, g1);
        }

        return min(f0[need], f1[need]);
    }

    int minOperations(vector<int>& nums, int k) {
        int n = (int)nums.size();
        if (k == 0) return 0;
        if (k > n / 2) return -1;

        vector<long long> w(n, 0);
        for (int i = 0; i < n; ++i) {
            long long L = nums[(i - 1 + n) % n];
            long long R = nums[(i + 1) % n];
            w[i] = max(0LL, max(L, R) + 1 - 1LL * nums[i]);
        }

        long long ans = INF;

        // 情况 1：不选 0
        ans = min(ans, solveLine(w, 1, n - 1, k));

        // 情况 2：选 0
        ans = min(ans, w[0] + solveLine(w, 2, n - 2, k - 1));

        return (ans >= INF / 2 ? -1 : (int)ans);
    }
};
```

**复杂度分析**

时间复杂度：$O(nk)$。空间复杂度：$O(k)$。

**变形**

若改为线性数组版本，只需保留一次链 DP；若改为“最大化峰值数后最小代价”，可先枚举可行峰值数再做同一套恰选 DP。
