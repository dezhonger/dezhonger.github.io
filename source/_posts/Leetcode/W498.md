## 3903 - Smallest Stable Index I

**题意**

题目链接：[3903. Smallest Stable Index I](https://leetcode.com/problems/smallest-stable-index-i/)

给定数组 `nums` 与整数 `k`。对每个下标 $i$ 定义不稳定值：

$$
score_i = \max(nums[0..i]) - \min(nums[i..n-1])
$$

要求返回满足 $score_i \le k$ 的最小下标；若不存在返回 `-1`。

**解题思路**

I 版数据范围较小（$n \le 100$），直接按定义枚举每个下标即可：

1. 扫描区间 $[0,i]$ 求前缀最大值。
2. 扫描区间 $[i,n-1]$ 求后缀最小值。
3. 若差值不超过 $k$，立即返回 $i$。

该写法完全贴合题意，实现最直接。

**代码**

```cpp
class Solution {
public:
    int firstStableIndex(vector<int>& nums, int k) {
        int n = (int)nums.size();
        for (int i = 0; i < n; ++i) {
            int mx = nums[0];
            for (int j = 1; j <= i; ++j) {
                mx = max(mx, nums[j]);
            }
            int mn = nums[i];
            for (int j = i + 1; j < n; ++j) {
                mn = min(mn, nums[j]);
            }
            if ((long long)mx - mn <= k) {
                return i;
            }
        }
        return -1;
    }
};
```

**复杂度分析**

每个下标各做两段线性扫描，总时间复杂度为 $O(n^2)$，空间复杂度为 $O(1)$。

**变形**

同一题意在 II 版把数据放大后，核心是将 $O(n^2)$ 优化到 $O(n)$。

## 3904 - Smallest Stable Index II

**题意**

题目链接：[3904. Smallest Stable Index II](https://leetcode.com/problems/smallest-stable-index-ii/)

定义与 I 版一致，但数据范围提升到 $n \le 10^5$，需要返回最小稳定下标。

**解题思路**

直接暴力会达到 $O(n^2)$，无法通过。使用前后缀预处理：

- 定义 $pre_i = \max(nums[0..i])$。
- 定义 $suf_i = \min(nums[i..n-1])$。

则位置 $i$ 的不稳定值为 $pre_i - suf_i$。从小到大扫描第一个满足
$pre_i - suf_i \le k$ 的下标即为答案。

正确性来自定义等价：$pre_i$ 与 $suf_i$ 分别精确对应题目的前缀最大、后缀最小。

**代码**

```cpp
class Solution {
public:
    int firstStableIndex(vector<int>& nums, int k) {
        int n = (int)nums.size();
        vector<int> pre(n), suf(n);

        pre[0] = nums[0];
        for (int i = 1; i < n; i++) {
            pre[i] = max(pre[i - 1], nums[i]);
        }

        suf[n - 1] = nums[n - 1];
        for (int i = n - 2; i >= 0; i--) {
            suf[i] = min(suf[i + 1], nums[i]);
        }

        for (int i = 0; i < n; i++) {
            if ((long long)pre[i] - suf[i] <= k) {
                return i;
            }
        }
        return -1;
    }
};
```

**复杂度分析**

预处理两次数组扫描，加一次线性判定，总时间复杂度 $O(n)$，空间复杂度 $O(n)$。

**变形**

1. 返回所有稳定下标：扫描时收集所有满足条件的位置。
2. 统计稳定下标个数：扫描时计数即可。
3. 若改为不重叠区间 $[0,i]$ 与 $[i+1,n-1]$，只需调整后缀最小值索引边界。

## 3905 - Multi Source Flood Fill

**题意**

题目链接：[3905. Multi Source Flood Fill](https://leetcode.com/problems/multi-source-flood-fill/)

给定 $n \times m$ 网格和多个初始有色源点。每轮所有已着色格子同时向四邻域的未着色格子扩散；若同一轮多个颜色同时到达同一格，取最大颜色。返回最终网格。

**解题思路**

这是多源 BFS。关键不在“是否可达”，而在“最早到达轮次 + 同轮冲突取最大色值”。

定义一维数组 $dist_p$ 表示格子 $p$ 的最早染色轮次，初始化为 $-1$。初始源点的轮次为 $0$ 并全部入队。

对队头格子 $(r,c)$，其邻居 $(nr,nc)$ 的候选到达轮次为 $dist_{r,c}+1$：

1. 若邻居未染色（$dist=-1$），则首次到达，记录轮次、赋色并入队。
2. 若邻居已在同一轮次被到达（$dist$ 等于候选轮次），说明发生“同轮冲突”，取颜色最大值。
3. 若邻居更早被到达，则忽略。

该过程严格对应题意的“同步扩散”。

**代码**

```cpp
class Solution {
public:
    vector<vector<int>> colorGrid(int n, int m, vector<vector<int>>& sources) {
        vector<vector<int>> ans(n, vector<int>(m, 0));
        vector<int> dist(n * m, -1);
        queue<int> q;

        for (const auto& s : sources) {
            int r = s[0], c = s[1], color = s[2];
            int idx = r * m + c;
            ans[r][c] = color;
            dist[idx] = 0;
            q.push(idx);
        }

        int dr[4] = {0, 0, 1, -1};
        int dc[4] = {1, -1, 0, 0};

        while (!q.empty()) {
            int currIdx = q.front();
            q.pop();

            int r = currIdx / m;
            int c = currIdx % m;
            int currColor = ans[r][c];
            int nextDist = dist[currIdx] + 1;

            for (int d = 0; d < 4; ++d) {
                int nr = r + dr[d];
                int nc = c + dc[d];
                if (nr < 0 || nr >= n || nc < 0 || nc >= m) {
                    continue;
                }

                int nIdx = nr * m + nc;
                if (dist[nIdx] == -1) {
                    dist[nIdx] = nextDist;
                    ans[nr][nc] = currColor;
                    q.push(nIdx);
                } else if (dist[nIdx] == nextDist) {
                    ans[nr][nc] = max(ans[nr][nc], currColor);
                }
            }
        }

        return ans;
    }
};
```

**复杂度分析**

设 $N = n \cdot m$。每个格子最多首次入队一次，每次处理固定 4 个方向，总时间复杂度 $O(N)$，空间复杂度 $O(N)$。

**变形**

1. 冲突改为取最小颜色：同轮更新由 `max` 改为 `min`。
2. 加入障碍格：扩展时跳过不可进入格子。
3. 不同颜色扩散速度不同：从 BFS 改为按时间最短路（通常是 Dijkstra）。

## 3906 - Count Good Integers on a Grid Path

**题意**

题目链接：[3906. Count Good Integers on a Grid Path](https://leetcode.com/problems/count-good-integers-on-a-grid-path/)

给定区间 $[l,r]$ 和长度为 6 的 `directions`（恰有 3 个 `D` 与 3 个 `R`）。

对每个整数 $x$：

1. 转成 16 位十进制串（不足补前导零）。
2. 按行优先填入 $4 \times 4$ 网格。
3. 从 $(0,0)$ 按 `directions` 走 6 步，共访问 7 个格子。
4. 取路径上 7 个数字，若该序列非递减，则 $x$ 为 good。

求区间内 good 整数个数。

**解题思路**

关键是把题目转成数位 DP。

先将路径映射到 16 位串下标：网格坐标 $(r,c)$ 对应下标 $id=4r+c$。由 `directions` 得到固定 7 个位置。

设 $F(x)$ 表示区间 $[0,x]$ 内 good 个数，则答案为：

$$
F(r)-F(l-1)
$$

对固定上界 $x$ 做 16 位数位 DP：

- 状态：$dfs(pos,tight,last)$
- 含义如下：
  - 位置 `pos`：当前处理到第几位（0 到 15）
  - `tight`：前缀是否贴合上界
  - `last`：上一条路径位上选择的数字

转移时枚举本位数字 $d$：

1. 若当前位置不在路径上，直接转移。
2. 若在路径上，需要满足非递减约束；首个路径位可直接记录，后续要求 $d \ge last$。

该 DP 只在 16 位上运行，状态规模很小。

关于记忆化：这里对 `tight=0/1` 都做记忆化也成立，因为每次计算 $F(x)$ 时上界串固定，且会重新清空缓存，不会跨上界复用。

**代码**

```cpp
class Solution {
public:
    string s;
    vector<int> vis;
    long long memo[20][2][11];
    bool ok[20][2][11];

    long long dfs(int p, int lim, int lst) {
        if (p == 16) {
            return 1;
        }
        if (ok[p][lim][lst]) {
            return memo[p][lim][lst];
        }
        ok[p][lim][lst] = true;

        long long res = 0;
        int up = lim ? (s[p] - '0') : 9;

        for (int d = 0; d <= up; d++) {
            int nl = lim && (d == up);
            if (vis[p]) {
                if (p == 0 || d >= lst) {
                    res += dfs(p + 1, nl, d);
                }
            } else {
                res += dfs(p + 1, nl, lst);
            }
        }

        memo[p][lim][lst] = res;
        return res;
    }

    long long calc(long long x, const string& dir) {
        if (x < 0) {
            return 0;
        }

        s = to_string(x);
        while ((int)s.size() < 16) {
            s = "0" + s;
        }

        vis.assign(16, 0);
        int r = 0, c = 0;
        vis[0] = 1;
        for (char ch : dir) {
            if (ch == 'D') {
                r++;
            } else {
                c++;
            }
            vis[r * 4 + c] = 1;
        }

        memset(ok, 0, sizeof(ok));
        return dfs(0, 1, 0);
    }

    long long countGoodIntegersOnPath(long long l, long long r, string directions) {
        return calc(r, directions) - calc(l - 1, directions);
    }
};
```

**复杂度分析**

位数固定 16，状态约为 $16 \times 2 \times 10$，每个状态最多枚举 10 个数字，时间复杂度可记为 $O(16 \cdot 2 \cdot 10 \cdot 10)$，空间复杂度为 $O(16 \cdot 2 \cdot 10)$。

**变形**

1. 改成严格递增：把约束从 $d \ge last$ 改为 $d > last$。
2. 改成路径数字和受限：在状态中增加路径和维度。
3. 改为其他进制：保持框架不变，调整每位枚举上界。
