## 3917 - Count Indices With Opposite Parity

**题意**

给定长度为 $n$ 的数组 `nums`，对每个下标 $i$，定义其得分为：满足 $i<j<n$ 且 `nums[i]` 与 `nums[j]` 奇偶性不同的下标 $j$ 的数量。返回长度为 $n$ 的数组 `answer`，其中 `answer[i]` 为下标 $i$ 的得分。

- $1 \le n \le 100$
- $1 \le nums_i \le 100$

**解题思路**

从右往左遍历，维护当前位置右侧已经出现的奇数个数 `odd` 和偶数个数 `eve`。

若 `nums[i]` 为奇数，则其得分为 `eve`（偶数在右侧不同奇偶）；若为偶数，则得分为 `odd`（奇数在右侧不同奇偶）。遍历结束后即得答案。

**代码**

```cpp
class Solution {
public:
    vector<int> countOppositeParity(vector<int>& nums) {
        int n = nums.size();
        vector<int> ans(n);
        int odd = 0, eve = 0;

        for (int i = n - 1; i >= 0; i--) {
            if (nums[i] & 1) {
                ans[i] = eve;
                odd++;
            } else {
                ans[i] = odd;
                eve++;
            }
        }

        return ans;
    }
};
```

**复杂度分析**

时间复杂度：$O(n)$，空间复杂度：$O(n)$（返回数组）。

## 3918 - Sum of Primes Between Number and Its Reverse

**题意**

给定整数 $n$，设 $r$ 为 $n$ 的十进制反转数。返回区间 $[\min(n,r),\max(n,r)]$ 内所有质数之和。

- $1 \le n \le 1000$

**解题思路**

反转整数后枚举区间内每个数，判断是否为质数并累加。质数判定只需检查 $2$ 到 $\sqrt{x}$ 是否有因子。

**代码**

```cpp
class Solution {
public:
    int reverse(int x) {
        int res = 0;
        while (x > 0) {
            res = res * 10 + x % 10;
            x /= 10;
        }
        return res;
    }

    bool isPrime(int x) {
        if (x < 2) return false;
        for (int i = 2; i * i <= x; i++) {
            if (x % i == 0) return false;
        }
        return true;
    }

    int sumOfPrimesInRange(int n) {
        int r = reverse(n);
        int l = min(n, r);
        int rr = max(n, r);

        int ans = 0;
        for (int i = l; i <= rr; i++) {
            if (isPrime(i)) {
                ans += i;
            }
        }

        return ans;
    }
};
```

**复杂度分析**

设区间长度为 $m = |n-r|+1$，最大数为 $V \le 1000$。时间复杂度 $O(m\sqrt{V})$，空间复杂度 $O(1)$。

## 3919 - Minimum Cost to Move Between Indices

**题意**

给定严格递增数组 `nums`，对每个下标 $x$，定义 `closest(x)` 为下标 $x$ 的相邻下标中与 `nums[x]` 数值差最小的下标；若两个相邻下标差值相同，选较小下标。

从下标 $x$ 可以：
1. 移动到任意下标 $y$，代价为 $|nums[x]-nums[y]|$；
2. 移动到 `closest(x)`，代价为 $1$。

给定 $queries$ 中每个询问 $[l_i,r_i]$，求从 $l_i$ 到 $r_i$ 的最小总代价。

- $2 \le nums.length \le 10^5$
- $1 \le queries.length \le 10^5$
- `nums` 严格递增

**解题思路**

由于 `nums` 严格递增，从 $l$ 到 $r$（$l<r$）的普通移动代价可拆为相邻差值之和：

$$
nums[r] - nums[l] = \sum_{k=l}^{r-1} (nums[k+1]-nums[k])
$$

而 `closest` 移动只发生在相邻下标之间，代价可从原来的相邻差值变为 $1$。

预处理两个方向的前缀和：

- `pre[i]`：从 $0$ 走到 $i$ 向右的最优代价之和
- `suf[i]`：从 $i$ 走到 $n-1$ 向左的最优代价之和

对于询问 $[l,r]$：
- 若 $l<r$，答案为 `pre[r]-pre[l]`
- 若 $l>r$，答案为 `suf[l]-suf[r]`

**代码**

```cpp
class Solution {
public:
    vector<int> minCost(vector<int>& nums, vector<vector<int>>& queries) {
        int n = nums.size();

        vector<int> clo(n);

        for (int i = 0; i < n; i++) {
            if (i == 0) {
                clo[i] = 1;
            } else if (i == n - 1) {
                clo[i] = n - 2;
            } else {
                int a = nums[i] - nums[i - 1];
                int b = nums[i + 1] - nums[i];
                if (a <= b) clo[i] = i - 1;
                else clo[i] = i + 1;
            }
        }

        vector<LL> pre(n), suf(n);

        for (int i = 0; i + 1 < n; i++) {
            LL c = nums[i + 1] - nums[i];
            if (clo[i] == i + 1) c = 1;
            pre[i + 1] = pre[i] + c;
        }

        for (int i = 1; i < n; i++) {
            LL c = nums[i] - nums[i - 1];
            if (clo[i] == i - 1) c = 1;
            suf[i] = suf[i - 1] + c;
        }

        vector<int> ans;

        for (auto& q : queries) {
            int l = q[0], r = q[1];

            if (l < r) {
                ans.push_back(pre[r] - pre[l]);
            } else {
                ans.push_back(suf[l] - suf[r]);
            }
        }

        return ans;
    }
};
```

**复杂度分析**

时间复杂度：$O(n+q)$，空间复杂度：$O(n)$。

**变形**

若 `nums` 不是递增数组，原前缀和做法失效，需要用最短路（Dijkstra）处理。按 `nums[i]` 排序后连 $O(n)$ 条无向边，再加 `closest` 有向边，总时间 $O(k\cdot n\log n+q)$，其中 $k$ 为不同起点数量。

## 3920 - Maximize Fixed Points After Deletions

**题意**

给定数组 `nums`，可以删除任意数量元素，删除后剩余元素保持相对顺序并从 $0$ 重新编号。位置 $i$ 为 fixed point 当且仅当删除后该位置满足 `nums[i]==i`。求最多可得到的 fixed point 数量。

- $1 \le nums.length \le 10^5$
- $0 \le nums_i \le 10^5$

**解题思路**

若原数组位置 $i$ 保留后在新的下标为 $nums[i]$，则其前面必须恰好保留 $nums[i]$ 个元素，因此必须有 $i\ge nums[i]$，这是位置 $i$ 能成为 fixed point 的必要条件。

设 $v=nums[i]$，$d=i-nums[i]$（$d\ge 0$）。若同时选择两个位置 $i<j$ 作为 fixed point，需满足 $v_i<v_j$（新下标严格递增）以及 $d_i\le d_j$（中间有足够元素填充）。于是问题转化为：

> 在所有满足 $i\ge nums[i]$ 的点中，求最长序列使得 $nums[i]$ 严格递增且 $i-nums[i]$ 非递减。

按 $v=nums[i]$ 排序后，对每个点查询所有 $d'\le d$ 的最大 dp 值并加 $1$ 更新。由于 dp 值只与 $d$ 相关，用 Fenwick 树维护前缀最大值即可。相同 $v$ 的一组点需先统一查询再统一更新，防止同值之间互相转移。

**代码**

```cpp
class Solution {
public:
    struct BIT {
        int n;
        vector<int> tr;

        BIT(int n) : n(n), tr(n + 1, 0) {}

        void update(int x, int v) {
            for (; x <= n; x += x & -x) {
                tr[x] = max(tr[x], v);
            }
        }

        int query(int x) {
            int res = 0;
            for (; x > 0; x -= x & -x) {
                res = max(res, tr[x]);
            }
            return res;
        }
    };

    int maxFixedPoints(vector<int>& nums) {
        int n = nums.size();

        vector<pair<int, int>> pts;

        for (int i = 0; i < n; i++) {
            if (i >= nums[i]) {
                int v = nums[i];
                int d = i - nums[i];
                pts.push_back({v, d});
            }
        }

        sort(pts.begin(), pts.end());

        BIT bit(n + 2);
        int ans = 0;

        for (int l = 0; l < (int)pts.size(); ) {
            int r = l;
            while (r < (int)pts.size() && pts[r].first == pts[l].first) {
                r++;
            }

            vector<pair<int, int>> tmp;

            for (int i = l; i < r; i++) {
                int d = pts[i].second;
                int dp = bit.query(d + 1) + 1;
                tmp.push_back({d, dp});
                ans = max(ans, dp);
            }

            for (auto [d, dp] : tmp) {
                bit.update(d + 1, dp);
            }

            l = r;
        }

        return ans;
    }
};
```

**复杂度分析**

时间复杂度：$O(n\log n)$，空间复杂度：$O(n)$。

**变形**

Fenwick 树中 `query(d+1)` 查询所有 $d'\le d$ 的最大 dp 值；`+1` 将当前点接入序列。若 $nums_i$ 范围较小，也可以按值分桶省掉排序。
