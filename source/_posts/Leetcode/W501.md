## A - Concatenate Array With Reverse

**题意**

题目链接：[Concatenate Array With Reverse](https://leetcode.com/problems/concatenate-array-with-reverse/description/)

给定一个长度为 $n$ 的整数数组 `nums`，构造一个长度为 $2n$ 的数组 `ans`：
- 前 $n$ 个元素与 `nums` 完全相同；
- 后 $n$ 个元素为 `nums` 的逆序。

即对 $0 \le i \le n-1$，有 $ans[i] = nums[i]$，$ans[i+n] = nums[n-i-1]$。

返回构造的数组 `ans`。

**解题思路**

直接模拟即可。首先按原顺序将 `nums` 的元素加入答案数组，再逆序遍历将元素加入答案数组。

**代码**

```cpp
class Solution {
public:
    vector<int> concatWithReverse(vector<int>& nums) {
        vector<int> ans;
        int n = nums.size();

        for (int i = 0; i < n; i++) {
            ans.push_back(nums[i]);
        }

        for (int i = n - 1; i >= 0; i--) {
            ans.push_back(nums[i]);
        }

        return ans;
    }
};
```

**复杂度分析**

设 $n = nums.length$。

时间复杂度：$O(n)$，遍历两次数组。

空间复杂度：$O(n)$，返回数组长度为 $2n$。若不计返回值，额外空间为 $O(1)$。

**变形**

1. 拼接数组自身 $k$ 次；
2. 拼接数组某个区间的逆序；
3. 原地修改数组实现相同功能。

## B - Count Valid Word Occurrences

**题意**

题目链接：[Count Valid Word Occurrences](https://leetcode.com/problems/count-valid-word-occurrences/description/)

给定字符串数组 `chunks`，按顺序拼接成完整字符串 `s`。

定义 **joiner hyphen**：若 `'-'` 的前后字符均存在且均为小写英文字母，则该 `-` 为 joiner hyphen。

定义 **word**：由小写英文字母和 joiner hyphen 组成的极大连续子串。其他字符均视为分隔符。

给定查询数组 `queries`，返回 `ans`，其中 $ans[i]$ 表示 $queries[i]$ 作为完整单词出现的次数。

**解题思路**

由于 `chunks` 总长度不超过 $10^5$，可直接拼接成完整字符串 `s`。

定义函数判断位置 $i$ 的字符是否可以属于单词：
1. 若 $s[i]$ 为小写字母，则可以；
2. 若 $s[i]$ 为 `'-'`，需满足 $i>0$、$i+1<n$、$s[i-1]$ 和 $s[i+1]$ 均为小写字母；
3. 否则为分隔符。

从左到右扫描 $s$：遇到分隔符跳过；遇到可属于单词的字符，扩展至分隔符为止，得到完整单词后用哈希表统计。最后对每个查询在哈希表中取出现次数。

**代码**

```cpp
class Solution {
public:
    // 统计拼接后字符串中，每个查询词作为完整单词出现的次数
    vector<int> countWordOccurrences(vector<string>& chunks, vector<string>& queries) {
        string s;
        int len = 0;

        for (string& x : chunks) {
            len += x.size();
        }

        s.reserve(len);

        for (string& x : chunks) {
            s += x;
        }

        int n = s.size();

        auto low = [&](char c) -> bool {
            return c >= 'a' && c <= 'z';
        };

        // 判断 s[i] 是否可以作为单词的一部分（valid character）
        auto ok = [&](int i) -> bool {
            if (low(s[i])) {
                return true;
            }

            if (s[i] != '-') {
                return false;
            }

            return i > 0 && i + 1 < n && low(s[i - 1]) && low(s[i + 1]);
        };

        unordered_map<string, int> mp;

        for (int i = 0; i < n; ) {
            if (!ok(i)) {
                i++;
                continue;
            }

            int j = i;
            string cur;

            while (j < n && ok(j)) {
                cur.push_back(s[j]);
                j++;
            }

            mp[cur]++;
            i = j;
        }

        vector<int> ans;
        ans.reserve(queries.size());

        for (string& q : queries) {
            ans.push_back(mp[q]);
        }

        return ans;
    }
};
```

**复杂度分析**

设 $N$ 为 `chunks` 拼接后的总长度，$M$ 为 `queries` 的总长度。

拼接字符串需要 $O(N)$，扫描提取所有单词需要 $O(N)$，所有查询需要 $O(M)$。

总时间复杂度：$O(N + M)$。

空间复杂度：$O(N + M)$，用于保存拼接后的字符串、哈希表及返回数组。

**变形**

1. 统计出现次数最多的 $k$ 个单词；
2. 支持插入新 `chunks` 后的增量更新；
3. 统计包含至少一个 joiner hyphen 的单词数量。

## C - Minimize Array Sum Using Divisible Replacements

**题意**

题目链接：[Minimize Array Sum Using Divisible Replacements](https://leetcode.com/problems/minimize-array-sum-using-divisible-replacements/description/)

给定整数数组 `nums`，可进行任意次操作：选择下标 $a, b$，满足 $nums[a] \% nums[b] == 0$，将 $nums[a]$ 替换为 $nums[b]$。

返回数组可能的最小元素和。

**解题思路**

关键观察：对于数 $x$，它最终只能变成数组中存在的、能整除 $x$ 的最小数字。

设 $d$ 为数组中出现的数，若 $d|x$，则 $x$ 可以被替换为 $d$。因此 $x$ 能达到的最小值就是数组中能整除 $x$ 的最小出现值。

用倍数枚举实现：统计数组中出现的每个值 $d$，然后对所有 $d$ 的倍数更新其最小可替换值。由于从小到大枚举，第一次更新到某个数时即为最小值。

正确性说明：
- 每次操作只能把 $x$ 变成某个数组中已存在的、整除 $x$ 的数；
- 所以 $x$ 的最终值不可能小于数组中能整除 $x$ 的最小值；
- 另一方面，若存在 $d|x$，可以直接用操作把 $x$ 替换为 $d$，故最小值可达。

**代码**

```cpp
const int INF = 1e9;

class Solution {
public:
    // 返回 nums 经过若干次合法替换后能得到的最小数组和
    long long minArraySum(vector<int>& nums) {
        int mx = 0;
        for (int x : nums) {
            mx = max(mx, x);
        }

        vector<int> fr(mx + 1, 0);
        for (int x : nums) {
            fr[x]++;
        }

        // bst[x] 表示 x 能被替换成的最小值（best replacement）
        vector<int> bst(mx + 1, INF);

        for (int d = 1; d <= mx; d++) {
            if (fr[d] == 0) {
                continue;
            }

            for (int m = d; m <= mx; m += d) {
                if (bst[m] == INF) {
                    bst[m] = d;
                }
            }
        }

        LL ans = 0;
        for (int x : nums) {
            ans += bst[x];
        }

        return ans;
    }
};
```

**复杂度分析**

设 $V = \max(nums)$。

枚举每个出现的数 $d$ 并遍历其倍数，时间复杂度为：

$$O(V \log V)$$

因为 $\sum_{d=1}^{V} \frac{V}{d} = O(V \log V)$。

空间复杂度：$O(V)$，用于 `fr` 和 `bst` 数组。

**变形**

1. 若 $nums[i]$ 扩大到 $10^9$，改用质因数分解枚举因子；
2. 求最大可能的数组和；
3. 限制操作次数，求最小可能的数组和。

## D - Minimum Cost to Buy Apples II

**题意**

题目链接：[Minimum Cost to Buy Apples II](https://leetcode.com/problems/minimum-cost-to-buy-apples-ii/description/)

有 $n$ 个商店，`prices[i]` 为商店 $i$ 的苹果价格。给定 $m$ 条无向道路 $roads[k] = [u, v, cost, tax]$：
- 空手经过道路，代价为 $cost$；
- 带苹果经过道路，代价为 $cost \times tax$。

从商店 $i$ 出发，可以：
1. 直接在 $i$ 买苹果；
2. 空手走到某商店 $j$ 买苹果；
3. 带苹果从 $j$ 返回 $i$。

去程和回程可以走不同路径。

返回数组 `ans`，其中 $ans[i]$ 表示从商店 $i$ 出发买苹果并返回的最小总花费。

**解题思路**

对于起点 $i$，若选择去商店 $j$ 买苹果，总花费为：

$$d_0(i,j) + d_1(i,j) + prices[j]$$

其中 $d_0$ 为空手图（边权 $cost$）的最短路，$d_1$ 为带苹果图（边权 $cost \times tax$）的最短路。

注意不能把每条边权设为 $cost + cost \times tax$ 后只跑一次最短路，因为去程和回程可以走不同路径。

由于 $n \le 1000$，$m \le 2000$，对每个起点 $i$ 分别在两张图上跑 Dijkstra，然后枚举所有买苹果的商店取最小值。

**代码**

```cpp
const LL INF = 4e18;

class Solution {
public:
    // 从起点 st 出发，在图 g 上跑 Dijkstra，返回到每个点的最短距离
    vector<LL> dij(int st, vector<vector<pair<int, LL>>>& g) {
        int n = g.size();
        vector<LL> dis(n, INF);
        priority_queue<pair<LL, int>, vector<pair<LL, int>>, greater<pair<LL, int>>> pq;

        dis[st] = 0;
        pq.push({0, st});

        while (!pq.empty()) {
            auto [d, u] = pq.top();
            pq.pop();

            if (d != dis[u]) {
                continue;
            }

            for (auto [v, w] : g[u]) {
                if (dis[v] > d + w) {
                    dis[v] = d + w;
                    pq.push({dis[v], v});
                }
            }
        }

        return dis;
    }

    // 返回从每个商店出发买苹果并返回的最小总花费
    vector<int> minCost(int n, vector<int>& prices, vector<vector<int>>& roads) {
        // g0: 空手图，边权为 cost
        // g1: 带苹果图，边权为 cost * tax
        vector<vector<pair<int, LL>>> g0(n), g1(n);

        for (auto& e : roads) {
            int u = e[0];
            int v = e[1];
            LL c = e[2];
            LL t = e[3];

            g0[u].push_back({v, c});
            g0[v].push_back({u, c});

            g1[u].push_back({v, c * t});
            g1[v].push_back({u, c * t});
        }

        vector<int> ans(n);

        for (int i = 0; i < n; i++) {
            vector<LL> d0 = dij(i, g0);
            vector<LL> d1 = dij(i, g1);

            LL res = INF;

            for (int j = 0; j < n; j++) {
                if (d0[j] == INF || d1[j] == INF) {
                    continue;
                }

                res = min(res, d0[j] + d1[j] + (LL)prices[j]);
            }

            ans[i] = (int)res;
        }

        return ans;
    }
};
```

**复杂度分析**

设 $n$ 为商店数量，$m$ 为道路数量。

每个起点需跑两次 Dijkstra，每次复杂度为 $O((n+m)\log n)$，总时间复杂度为 $O(n(n+m)\log n + n^2)$。

空间复杂度：$O(n + m)$。
