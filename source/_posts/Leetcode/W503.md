## A - Limit Occurrences in Sorted Array

**题意**

题目链接：[Limit Occurrences in Sorted Array](https://leetcode.com/problems/limit-occurrences-in-sorted-array/description/)

给定一个按非降序排序的整数数组 `nums` 和整数 `k`。返回一个新数组，使得每个不同元素最多出现 `k` 次，并保持原相对顺序。若某个元素原本出现次数不少于 `k`，则结果中必须恰好保留 `k` 次。

- `1 <= nums.length <= 100`
- `1 <= nums[i] <= 100`
- `nums` 已排序
- `1 <= k <= nums.length`

**解题思路**

由于数组已经排序，相同元素必然连续出现。顺次扫描时，只需维护当前连续段已经出现的次数 `cnt`。当 `nums[i]` 与前一个元素不同，说明开始了新的连续段，令 `cnt = 1`；否则令 `cnt++`。只有 `cnt <= k` 时才把当前元素加入答案。

**代码**

```cpp
class Solution {
public:
    vector<int> limitOccurrences(vector<int>& nums, int k) {
        vector<int> ans;
        int cnt = 0; // count，当前连续段已经出现的次数

        for (int i = 0; i < (int)nums.size(); i++) {
            if (i == 0 || nums[i] != nums[i - 1]) {
                cnt = 1;
            } else {
                cnt++;
            }

            if (cnt <= k) {
                ans.push_back(nums[i]);
            }
        }

        return ans;
    }
};
```

**复杂度分析**

时间复杂度：$O(n)$，只需遍历一遍数组。

空间复杂度：$O(n)$，答案数组长度最多为 $n$。

**变形**

如果要做原地版本，可以把答案直接写回 `nums`，再 `resize` 到写入位置。由于 `nums` 已排序，判断条件可以写成 `p < k || nums[p - k] != x`，这就是 LeetCode 80 的同类思路。

## B - Password Strength

**题意**

题目链接：[Password Strength](https://leetcode.com/problems/password-strength/description/)

给定字符串 `password`。按如下规则计算强度：

- 每个不同的小写字母贡献 `1` 分；
- 每个不同的大写字母贡献 `2` 分；
- 每个不同的数字贡献 `3` 分；
- 每个不同的特殊字符 `"!@#$"` 贡献 `5` 分。

每个字符最多只计分一次，重复出现不重复计分。返回总分。

- `1 <= password.length <= 10^5`
- `password` 只包含大小写字母、数字和 `"!@#$"`

**解题思路**

每个字符只按“是否出现过”计分，所以只需记录每个 ASCII 字符是否已经见过。字符集很小，直接用大小为 `128` 的标记数组即可。

遍历字符串时：
- 若字符已经出现过，跳过；
- 否则根据字符类别累加对应分值，并标记已出现。

**代码**

```cpp
class Solution {
public:
    int passwordStrength(string password) {
        vector<int> vis(128, 0); // visited，字符是否出现过
        int ans = 0;

        for (char c : password) {
            if (vis[c]) {
                continue;
            }
            vis[c] = 1;

            if ('a' <= c && c <= 'z') {
                ans += 1;
            } else if ('A' <= c && c <= 'Z') {
                ans += 2;
            } else if ('0' <= c && c <= '9') {
                ans += 3;
            } else {
                ans += 5;
            }
        }

        return ans;
    }
};
```

**复杂度分析**

时间复杂度：$O(n)$。

空间复杂度：$O(1)$，标记数组大小固定。

**变形**

如果字符集扩大但仍然很小，仍然可以用标记数组。若字符集变大，再改成 `unordered_set<char>` 或位运算位图即可。

## C - Minimum Operations to Sort a Permutation

**题意**

题目链接：[Minimum Operations to Sort a Permutation](https://leetcode.com/problems/minimum-operations-to-sort-a-permutation/description/)

给定一个长度为 `n` 的排列 `nums`，元素恰好是 `0 ~ n - 1`。允许的操作只有两种：

- 翻转整个数组；
- 将数组左移一位。

返回把数组变成升序 `[0, 1, 2, ..., n - 1]` 的最少操作次数；若无法做到，返回 `-1`。

- `1 <= n <= 10^5`
- `nums` 是 `0 ~ n - 1` 的一个排列

**解题思路**

允许的操作只会改变数组的整体位置或整体方向，所以从有序数组出发，能到达的状态只有两类：

- 递增循环数组；
- 递减循环数组。

因此只要判断 `nums` 是否属于这两类之一，再计算最少操作数即可。

判断递增循环数组：检查是否对所有位置都有
$$
nums[(i+1)\bmod n] = (nums[i] + 1)\bmod n
$$
若成立，设 `nums[0] = p`。把它恢复成有序数组有两种方式：
- 直接左移 `n - p` 次；
- 先翻转，再左移 `p` 次，再翻转，共 `p + 2` 次。
取最小值即可。

判断递减循环数组：检查是否对所有位置都有
$$
nums[(i+1)\bmod n] = (nums[i] - 1 + n)\bmod n
$$
若成立，设它是从逆序数组左移 `p` 次得到的。恢复成有序数组有两种方式：
- 先翻转，再左移 `p` 次，共 `p + 1` 次；
- 先左移 `n-p` 次，再翻转，共 `n-p+1` 次。
取最小值即可。

**代码**

```cpp
class Solution {
public:
    int minOperations(vector<int>& nums) {
        int n = nums.size();
        int ans = (int)1e9;

        bool inc = true; // increasing cycle，递增循环数组
        bool dec = true; // decreasing cycle，递减循环数组

        for (int i = 0; i < n; i++) {
            int j = (i + 1) % n;

            if (nums[j] != (nums[i] + 1) % n) {
                inc = false;
            }

            if (nums[j] != (nums[i] - 1 + n) % n) {
                dec = false;
            }
        }

        if (inc) {
            int p = nums[0]; // pivot，递增循环数组的起点值
            int q = (n - p) % n; // shift，直接左移的次数
            ans = min(ans, min(q, p + 2));
        }

        if (dec) {
            int p = (n - 1 - nums[0] + n) % n; // shift，逆序数组左移了多少次
            ans = min(ans, min(p + 1, n - p + 1));
        }

        if (ans == (int)1e9) {
            return -1;
        }
        return ans;
    }
};
```

**复杂度分析**

时间复杂度：$O(n)$。

空间复杂度：$O(1)$。

**变形**

如果允许的操作再加上“右移一位”，本质上仍然只是在同一个循环群里移动，答案依旧可以通过判断循环形态后直接计算。

## D - Number of Pairs After Increment

**题意**

题目链接：[Number of Pairs After Increment](https://leetcode.com/problems/number-of-pairs-after-increment/description/)

给定数组 `nums1`、`nums2` 和查询数组 `queries`。有两种操作：

- `[1, x, y, val]`：给 `nums2[x..y]` 每个元素都加上 `val`；
- `[2, tot]`：询问有多少对 `(j, k)` 满足 `nums1[j] + nums2[k] == tot`。

返回所有类型 2 询问的答案。

- `1 <= nums1.length <= 5`
- `1 <= nums2.length <= 5 * 10^4`
- `1 <= queries.length <= 5 * 10^4`
- `queries[i]` 只可能是上述两种格式之一

**解题思路**

因为 `nums1.length <= 5`，每次查询时枚举 `nums1` 的每个元素即可。固定 `nums1[j]` 后，问题变成：当前 `nums2` 中有多少个元素等于 `tot - nums1[j]`。

所以关键在于动态维护 `nums2`：
- 支持区间加；
- 支持查询某个值在整个数组中的出现次数。

这正适合分块。把 `nums2` 切成若干块，每块维护：
- `a[i]`：块内元素的基础值；
- `lz[b]`：块的懒标记；
- `cnt[b]`：块内基础值出现次数。

真实值为 `a[i] + lz[b]`。

区间加时：
- 整块覆盖的部分只加懒标记；
- 两端零散部分直接改基础值并重建该块频率表。

查询某个值 `x` 出现次数时，对每个块 `b`，只要统计 `a[i] == x - lz[b]` 的个数，也就是直接查 `cnt[b][x - lz[b]]`。

`bs / bn / lz / le / ri / reb` 这些短变量含义如下：
- `bs`：block size，块大小；
- `bn`：block number，块数量；
- `lz`：lazy tag，懒标记；
- `le`：left end，每个块的左端点；
- `ri`：right end，每个块的右端点；
- `reb`：rebuild，重建块频率表。

**代码**

```cpp
using LL = long long;

class Solution {
private:
    int n;
    int bs; // block size，块大小
    int bn; // block number，块数量
    vector<LL> a; // base array，块内基础值
    vector<LL> lz; // lazy tag，懒标记
    vector<int> le; // left end，每个块的左端点
    vector<int> ri; // right end，每个块的右端点
    vector<unordered_map<LL, int>> cnt; // count，块内基础值频率表

    void reb(int b) { // rebuild，重建第 b 个块
        cnt[b].clear();
        for (int i = le[b]; i <= ri[b]; i++) {
            cnt[b][a[i]]++;
        }
    }

    void add(int l, int r, LL v) {
        int x = l / bs;
        int y = r / bs;

        if (x == y) {
            for (int i = l; i <= r; i++) {
                a[i] += v;
            }
            reb(x);
            return;
        }

        for (int i = l; i <= ri[x]; i++) {
            a[i] += v;
        }
        reb(x);

        for (int i = le[y]; i <= r; i++) {
            a[i] += v;
        }
        reb(y);

        for (int b = x + 1; b <= y - 1; b++) {
            lz[b] += v;
        }
    }

    int qry(LL x) {
        int res = 0;

        for (int b = 0; b < bn; b++) {
            LL need = x - lz[b];
            auto it = cnt[b].find(need);
            if (it != cnt[b].end()) {
                res += it->second;
            }
        }

        return res;
    }

public:
    vector<int> numberOfPairs(vector<int>& nums1, vector<int>& nums2, vector<vector<int>>& queries) {
        n = nums2.size();
        bs = 320;
        bn = (n + bs - 1) / bs;

        a.assign(n, 0);
        for (int i = 0; i < n; i++) {
            a[i] = nums2[i];
        }

        lz.assign(bn, 0);
        le.assign(bn, 0);
        ri.assign(bn, 0);
        cnt.assign(bn, unordered_map<LL, int>());

        for (int b = 0; b < bn; b++) {
            le[b] = b * bs;
            ri[b] = min(n - 1, (b + 1) * bs - 1);
            cnt[b].reserve(bs * 2);
            cnt[b].max_load_factor(0.7);
            reb(b);
        }

        vector<int> ans;

        for (auto &q : queries) {
            if (q[0] == 1) {
                int x = q[1];
                int y = q[2];
                int v = q[3];
                add(x, y, v);
            } else {
                LL tot = q[1];
                int res = 0;

                for (int x : nums1) {
                    res += qry(tot - x);
                }

                ans.push_back(res);
            }
        }

        return ans;
    }
};
```

**复杂度分析**

设 `n = nums2.length`，`q = queries.length`，块大小为 `B`，块数量约为 $n/B$。

一次区间加复杂度约为 $O(B + n/B)$，一次值查询复杂度约为 $O(n/B)$。因为 `nums1.length <= 5`，所以一次类型 2 查询仍是 $O(n/B)$。

总时间复杂度：$O(q(B + n/B))$，当 $B \approx \sqrt n$ 时，可看作 $O(q\sqrt n)$。

空间复杂度：$O(n)$。

**变形**

如果把 `nums1.length` 放大到 `10^5`，那么枚举 `nums1` 就不再合适，需要进一步把 `nums1` 也转成频率表或分块结构。
