## A - Digit Frequency Score

**题意**

题目链接：[Digit Frequency Score](https://leetcode.com/problems/digit-frequency-score/description/)

给定一个正整数 `n`。若数字 `d` 在 `n` 的十进制表示中出现了 `freq[d]` 次，则它对分数的贡献为 `d * freq[d]`。返回所有数字的贡献之和。

- `1 <= n <= 10^9`

**解题思路**

按照定义统计每个数字的出现次数后再计算贡献当然可以，但还有一个更直接的观察：

$$
\sum_{d=0}^{9} d \cdot freq[d]
$$

恰好就是 `n` 的所有十进制数位之和。因此不断取出 `n % 10` 累加，再令 `n /= 10` 删除最低位即可。

**代码**

```cpp
class Solution {
public:
    int digitFrequencyScore(int n) {
        int ans = 0;
        while (n > 0) {
            ans += n % 10;
            n /= 10;
        }
        return ans;
    }
};
```

**复杂度分析**

设 `n` 的十进制位数为 $D$。

时间复杂度：$O(D)$。

空间复杂度：$O(1)$。

**变形**

如果每个数字 `d` 的贡献改成 `weight[d] * freq[d]`，只需在取出当前数位 `d` 后累加 `weight[d]`。如果需要回答大量询问，可以预处理数位 DP，统计区间内所有数字的数位贡献。

## B - Maximum Number of Items From Sale I

**题意**

题目链接：[Maximum Number of Items From Sale I](https://leetcode.com/problems/maximum-number-of-items-from-sale-i/description/)

给定 `items` 和预算 `budget`，其中 `items[i] = [factor[i], price[i]]`。每种物品都有无限个副本，可以购买任意数量的物品，但总花费不能超过预算。

只要购买了至少一个物品 `i`，就会免费获得所有满足以下条件的物品 `j` 各一个：

- `i != j`；
- `factor[i]` 能整除 `factor[j]`。

同一种物品 `i` 买多个副本不会重复触发它的赠品，但同一个物品 `j` 可以由不同的 `i` 分别赠送。求购买副本数与免费副本数之和的最大值。

- `1 <= items.length <= 1000`
- `1 <= factor[i], price[i] <= 1500`
- `1 <= budget <= 1500`

**解题思路**

先计算 `cnt[i]`：第一次购买物品 `i` 时能获得多少个免费副本，即满足 `j != i` 且 `factor[j] % factor[i] == 0` 的物品数量。

对于每种物品 `i`，购买行为可以拆成两部分：

- 第一个副本花费 `price[i]`，得到 `1 + cnt[i]` 个物品。这一部分每种物品最多选择一次，是 0/1 背包；
- 后续每个副本仍花费 `price[i]`，但只能得到购买的这一个物品。这一部分可以选择无限次，是完全背包。

令 `dp[b]` 表示花费不超过 `b` 时最多能得到的物品数。处理物品 `i` 时：

1. 先倒序枚举预算，加入“第一次购买”的 `1 + cnt[i]` 收益，保证该奖励只使用一次；
2. 再正序枚举预算，加入额外副本的 `1` 收益，使当前物品可以继续购买任意次。

这个转移顺序把 0/1 背包和完全背包组合在了一起。

**代码**

```cpp
class Solution {
public:
    int maximumSaleItems(vector<vector<int>>& items, int budget) {
        int n = items.size();

        vector<int> fac(n), pri(n), cnt(n, 0);

        for (int i = 0; i < n; i++) {
            fac[i] = items[i][0];
            pri[i] = items[i][1];
        }

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (i != j && fac[j] % fac[i] == 0) {
                    cnt[i]++;
                }
            }
        }

        vector<int> dp(budget + 1, 0);

        for (int i = 0; i < n; i++) {
            int c = pri[i];

            // 0/1 背包：购买第一个副本，触发赠品奖励
            for (int b = budget; b >= c; b--) {
                dp[b] = max(dp[b], dp[b - c] + 1 + cnt[i]);
            }

            // 完全背包：继续购买该物品的额外副本
            for (int b = c; b <= budget; b++) {
                dp[b] = max(dp[b], dp[b - c] + 1);
            }
        }

        int ans = 0;
        for (int b = 0; b <= budget; b++) {
            ans = max(ans, dp[b]);
        }

        return ans;
    }
};
```

**复杂度分析**

设 $n = items.length$，$B = budget$。

计算每种物品的赠品数量需要 $O(n^2)$，混合背包需要 $O(nB)$。

总时间复杂度：$O(n^2 + nB)$。

空间复杂度：$O(n + B)$。

**变形**

如果购买第 `k` 个副本也会触发一次额外奖励，可以把每种物品拆成若干个有限次数的奖励段，再使用多重背包。若预算扩大到 $10^9$，就不能继续按预算做 DP，需要转而分析每次购买的边际收益。

## C - Maximum Number of Items From Sale II

**题意**

题目链接：[Maximum Number of Items From Sale II](https://leetcode.com/problems/maximum-number-of-items-from-sale-ii/description/)

仍然给定 `items[i] = [factor[i], price[i]]` 和预算 `budget`，每种物品都有无限个副本。

每购买一个物品 `i`，最多可以免费获得一个满足 `i != j` 且 `factor[i]` 能整除 `factor[j]` 的物品 `j`。对于固定的有序对 `(i, j)`，赠送关系最多使用一次；同一个 `j` 仍然可以由不同的 `i` 分别赠送。

求总花费不超过 `budget` 时，购买副本数与免费副本数之和的最大值。

- `1 <= items.length <= 10^5`
- `1 <= factor[i] <= items.length`
- `1 <= price[i] <= 10^9`
- `1 <= budget <= 10^9`

**解题思路**

### 1. 把每次购买看成边际收益

设

$$
deg_i = |\{j \mid j \ne i,\ factor_i \mid factor_j\}|
$$

对于物品 `i`：

- 前 `deg[i]` 个购买副本都可以分别匹配一个尚未使用的 `j`，所以每个副本的收益是 `2`：一个购买副本和一个免费副本；
- 再继续购买时已经没有新的有序对可用，每个副本的收益退化为 `1`。

因此原问题被拆成：

- 每种物品 `i` 有 `deg[i]` 个有限的“价值 2、费用 `price[i]`”购买机会；
- 另外还有无限个“价值 1、费用 `price[i]`”购买机会。

### 2. 价值 1 的副本只买最便宜的物品

设所有物品的最低价格为 `mn`。所有价值 1 的购买完全等价，显然只需购买价格为 `mn` 的物品。

对于一个价值 2 的购买机会，如果 `price[i] >= 2 * mn`，用相同或更少的钱直接购买两个最便宜的普通副本，收益也能达到 `2`，所以这样的机会没有必要保留。

只需收集满足以下条件的物品：

- `deg[i] > 0`；
- `price[i] < 2 * mn`。

每个物品贡献 `deg[i]` 个价格相同的价值 2 机会。把这些批次按价格从小到大排序，优先购买更便宜的机会；对于当前批次，一次购买 `min(deg[i], 剩余预算 / price[i])` 个。最后用剩余预算购买价格为 `mn` 的价值 1 副本。

若一共购买了 `num` 个价值 2 的机会，花费为 `cost`，答案就是

$$
2 \cdot num + \left\lfloor\frac{budget-cost}{mn}\right\rfloor
$$

### 3. 调和级数枚举整除关系

因为 `factor[i] <= n`，先统计每个因子值出现的次数 `cnt[x]`。再对每个 `d` 枚举它的所有倍数：

$$
mul[d] = \sum_{d \mid x} cnt[x]
$$

于是 `deg[i] = mul[factor[i]] - 1`，减去的 `1` 是物品 `i` 自己。

**代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

#ifdef LOCAL
#include "debug.h"
#else
#define debug(...) 42
#endif
using LL = long long;
using ULL = unsigned long long;

class Solution {
public:
    int maximumSaleItems(vector<vector<int>>& items, int budget) {
        int n = items.size();

        vector<int> fac(n), pri(n);
        vector<int> cnt(n + 1, 0);

        LL mn = (LL)4e18;

        for (int i = 0; i < n; i++) {
            fac[i] = items[i][0];
            pri[i] = items[i][1];
            cnt[fac[i]]++;
            mn = min(mn, (LL)pri[i]);
        }

        vector<int> mul(n + 1, 0);

        for (int d = 1; d <= n; d++) {
            for (int x = d; x <= n; x += d) {
                mul[d] += cnt[x];
            }
        }

        vector<pair<LL, LL>> vec;

        for (int i = 0; i < n; i++) {
            LL deg = mul[fac[i]] - 1;
            LL p = pri[i];

            if (deg > 0 && p < 2 * mn) {
                vec.push_back({p, deg});
            }
        }

        sort(vec.begin(), vec.end());

        LL bud = budget;
        LL cos = 0;
        LL num = 0;

        for (auto [p, deg] : vec) {
            if (cos + p > bud) {
                break;
            }

            LL can = (bud - cos) / p;
            LL tk = min(deg, can);

            cos += tk * p;
            num += tk;
        }

        LL ans = 2 * num + (bud - cos) / mn;

        return (int)ans;
    }
};
```

**复杂度分析**

设 $n = items.length$。

枚举所有 `d` 的倍数次数为

$$
\sum_{d=1}^{n}\left\lfloor\frac{n}{d}\right\rfloor = O(n\log n)
$$

候选批次排序同样需要 $O(n\log n)$。

总时间复杂度：$O(n\log n)$。

空间复杂度：$O(n)$。

**变形**

如果每个免费副本的价值不同，所有价值 2 的机会就不再等价，不能只按购买价格排序，需要把赠送关系建成带权匹配或流问题。如果每种物品还限制购买数量，则价值 1 的兜底购买也要改成按价格合并多个有限批次。

## D - Lexicographically Maximum MEX Array

**题意**

题目链接：[Lexicographically Maximum MEX Array](https://leetcode.com/problems/lexicographically-maximum-mex-array/description/)

给定整数数组 `nums`。每次选择当前数组的一个非空前缀，将这个前缀的 MEX 加入 `result`，再删除该前缀。重复操作直到 `nums` 为空。

MEX 是数组中没有出现的最小非负整数。返回所有可能的 `result` 中字典序最大的一个。

- `1 <= nums.length <= 10^5`
- `0 <= nums[i] <= 10^5`

**解题思路**

设当前剩余后缀的 MEX 为 `mex`。

### 1. 下一项的最大值就是整个后缀的 MEX

由于 `mex` 在整个剩余后缀中都不存在，它当然也不会出现在任何前缀中，所以任意前缀的 MEX 都不可能大于 `mex`。

另一方面，只要选择一个同时包含 `0, 1, ..., mex - 1` 的前缀，这个前缀的 MEX 就恰好是 `mex`。因此答案的下一项应该贪心地取当前后缀的 MEX。

为了给后面的答案保留尽可能多的元素，应选择包含 `0 ~ mex - 1` 的最短前缀。代码用 `vis` 和时间戳记录当前段已经出现过哪些必要数字，当 `need` 降到 `0` 时立即结束这一段。

### 2. 动态维护剩余后缀的 MEX

先用 `cnt[x]` 统计每个数在当前后缀中的出现次数。删除元素 `x` 时令 `cnt[x]--`；若它变成 `0`，说明 `x` 从剩余后缀中消失，新的 MEX 可能变小，因此更新：

```cpp
mex = min(mex, x);
```

删除元素只会让某些数字消失，不会让原本缺失的数字重新出现，所以后缀 MEX 只会不变或变小。

### 3. `mex == 0` 的特殊情况

若当前后缀的 MEX 是 `0`，说明剩余部分完全没有 `0`，任意非空前缀的 MEX 都是 `0`。为了让结果在公共前缀相同的情况下尽可能长，最优做法是每次只删除一个元素，把剩余每个元素都变成答案中的一个 `0`。

**代码**

```cpp
class Solution {
public:
    vector<int> maximumMEX(vector<int>& nums) {
        int n = nums.size();

        vector<int> cnt(n + 1, 0);
        for (int x : nums) {
            if (x <= n) {
                cnt[x]++;
            }
        }

        int mex = 0;
        while (mex <= n && cnt[mex] > 0) {
            mex++;
        }

        vector<int> ans;
        vector<int> vis(n + 1, 0);

        int time = 0;
        int i = 0;

        while (i < n) {
            int cur = mex;

            if (cur == 0) {
                while (i < n) {
                    ans.push_back(0);
                    i++;
                }
                break;
            }

            ans.push_back(cur);

            time++;
            int need = cur;

            while (i < n && need > 0) {
                int x = nums[i];

                if (x <= n) {
                    cnt[x]--;
                    if (cnt[x] == 0) {
                        mex = min(mex, x);
                    }
                }

                if (x < cur && vis[x] != time) {
                    vis[x] = time;
                    need--;
                }

                i++;
            }
        }

        return ans;
    }
};
```

**复杂度分析**

每个元素只会在统计频率时访问一次，并在所属前缀被删除时再访问一次。

时间复杂度：$O(n)$。

空间复杂度：$O(n)$。

**变形**

如果要求字典序最小的 MEX 数组，局部目标会完全相反，需要优先制造尽可能小的 MEX；例如当前首元素不是 `0` 时，可以立即得到 `0`。如果操作允许选择任意子数组而不是前缀，则剩余部分不再保持单一后缀结构，需要使用更复杂的区间数据结构维护可选段。
