## 3954 - Sum of Compatible Numbers in Range I

**题意**

题目链接：[Sum of Compatible Numbers in Range I](https://leetcode.com/problems/sum-of-compatible-numbers-in-range-i/description/)

给定两个整数 `n` 和 `k`。若正整数 `x` 同时满足：

- `abs(n - x) <= k`；
- `(n & x) == 0`；

则称 `x` 与 `n` 兼容。求所有兼容正整数 `x` 的总和。

- `1 <= n <= 100`
- `1 <= k <= 100`

**解题思路**

由绝对值条件：

$$
|n-x|\le k
$$

可以直接得到 `x` 的取值范围：

$$
n-k\le x\le n+k
$$

又因为 `x` 必须为正整数，所以枚举下界应为 `max(1, n - k)`。

对区间中的每个 `x` 检查 `(n & x) == 0`。按位与结果为 `0` 表示 `n` 和 `x` 的二进制表示不存在同时为 `1` 的位置，满足条件时将 `x` 加入答案即可。

**代码**

```cpp
class Solution {
public:
    int sumOfGoodIntegers(int n, int k) {
        int ans = 0;
        int l = max(1, n - k);
        int r = n + k;

        for (int x = l; x <= r; x++) {
            if ((n & x) == 0) {
                ans += x;
            }
        }

        return ans;
    }
};
```

**复杂度分析**

枚举区间最多包含 $2k+1$ 个整数。

时间复杂度：$O(k)$。

空间复杂度：$O(1)$。

**变形**

如果 `n` 和 `k` 扩大到 $10^{18}$，就不能继续枚举整个区间。可以分别计算区间 $[0,n+k]$ 与 $[0,n-k-1]$ 中满足 `(n & x) == 0` 的数字总和，再用数位 DP 或按位贡献法求差。

## 3955 - Valid Binary Strings With Cost Limit

**题意**

题目链接：[Valid Binary Strings With Cost Limit](https://leetcode.com/problems/valid-binary-strings-with-cost-limit/description/)

长度为 `n` 的二进制字符串 `s` 的代价定义为所有满足 `s[i] == '1'` 的下标 `i` 之和，下标从 `0` 开始。

若一个二进制字符串同时满足以下条件，则称它合法：

- 不包含连续的两个 `'1'`；
- 代价不超过 `k`。

返回所有长度为 `n` 的合法二进制字符串，顺序任意。

- `1 <= n <= 12`
- `0 <= k <= n * (n - 1) / 2`

**解题思路**

从左到右构造字符串。DFS 状态记录：

- `i`：当前准备填写的位置；
- `c`：已经放置的所有 `'1'` 的下标之和；
- `pre`：前一个字符是否为 `'1'`。

在位置 `i`：

1. 始终可以填写 `'0'`，代价和前一位状态分别变为 `c` 和 `0`；
2. 只有当 `pre == 0` 且 `c + i <= k` 时才能填写 `'1'`。

第二个分支同时完成了两种剪枝：`!pre` 保证没有连续的 `'1'`，`c + i <= k` 保证代价不会超出限制。因为后续新增代价均为非负数，一旦当前代价超过 `k`，该分支不可能重新变得合法。

当 `i == n` 时，当前字符串已经满足所有约束，加入答案。

**代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

#ifdef LOCAL
#include "debug.h"
#else
#define debug(...) 42
#endif
using LL = long long;
using ULL = unsigned long long;

class Solution {
    int nn, kk;
    string cur;
    vector<string> ans;

    void dfs(int i, int c, int pre) {
        if (i == nn) {
            ans.push_back(cur);
            return;
        }

        cur.push_back('0');
        dfs(i + 1, c, 0);
        cur.pop_back();

        if (!pre && c + i <= kk) {
            cur.push_back('1');
            dfs(i + 1, c + i, 1);
            cur.pop_back();
        }
    }

public:
    vector<string> generateValidStrings(int n, int k) {
        nn = n;
        kk = k;
        cur.clear();
        ans.clear();

        dfs(0, 0, 0);

        return ans;
    }
};
```

**复杂度分析**

忽略代价限制时，不含连续 `'1'` 的长度为 `n` 的字符串数量为 $F_{n+2}$，其中 $F$ 为斐波那契数列。保存每个答案需要复制长度为 `n` 的字符串。

时间复杂度：$O(nF_{n+2})$，也可以写成较宽松的 $O(n2^n)$。

除输出结果外，递归栈和当前字符串的空间复杂度为 $O(n)$；若计入输出，空间复杂度为 $O(nF_{n+2})$。

**变形**

如果只要求合法字符串的数量，可以定义 `dp[i][c][last]`，表示处理到位置 `i`、当前代价为 `c`、最后一位为 `last` 时的方案数，不再保存具体字符串。如果 `n` 较大但 `k` 较小，也可以只保留代价不超过 `k` 的状态。

## 3956 - Maximum Sum of M Non-Overlapping Subarrays I

**题意**

题目链接：[Maximum Sum of M Non-Overlapping Subarrays I](https://leetcode.com/problems/maximum-sum-of-m-non-overlapping-subarrays-i/description/)

给定长度为 `n` 的整数数组 `nums`，以及整数 `m`、`l`、`r`。需要选择至少一个、至多 `m` 个互不重叠的连续子数组，每个子数组的长度都在 `[l, r]` 内，最大化所有被选子数组的元素总和。

- `1 <= n <= 1000`
- `-10^9 <= nums[i] <= 10^9`
- `1 <= m <= n`
- `1 <= l <= r <= n`

题目要求至少选择一个子数组，因此即使所有元素均为负数，也不能返回 `0`。

**解题思路**

### 1. 前缀和与分层 DP

定义前缀和：

$$
pre[i]=\sum_{t=0}^{i-1}nums[t]
$$

则半开区间 `(j, i]`，即原数组下标 `[j, i - 1]` 的元素和为：

$$
pre[i]-pre[j]
$$

定义 $dp_k[i]$ 表示只考虑前 `i` 个元素，恰好选择 `k` 个互不重叠子数组时的最大总和。

考虑第 `k` 个子数组是否以位置 `i` 结束：

- 不使用第 `i` 个元素，答案为 $dp_k[i-1]$；
- 最后一个子数组为 `(j, i]`。由于其长度在 `[l, r]` 内，必须满足：

$$
i-r\le j\le i-l
$$

此时转移为：

$$
dp_k[i]
=\max\left(
dp_k[i-1],
\max_{j\in[i-r,i-l]}
\left(dp_{k-1}[j]+pre[i]-pre[j]\right)
\right)
$$

整理后得到：

$$
dp_k[i]
=\max\left(
dp_k[i-1],
pre[i]+\max_{j\in[i-r,i-l]}
\left(dp_{k-1}[j]-pre[j]\right)
\right)
$$

### 2. 单调队列维护滑动窗口最大值

当 `i` 从左向右移动时，合法的 `j` 始终构成滑动窗口 `[i-r, i-l]`。使用单调队列维护窗口中的：

$$
dp_{k-1}[j]-pre[j]
$$

队首就是当前窗口最大值。每个下标在每一层最多进入和离开队列各一次，因此每层 DP 为 $O(n)$。

初始化 $dp_0[i]=0$，表示在任意前缀中选择零个子数组的总和为 `0`。其他层初始化为负无穷，避免不存在的状态参与转移。

最多能选择的子数组数量为：

$$
K=\min\left(m,\left\lfloor\frac{n}{l}\right\rfloor\right)
$$

逐层计算 `1..K`，每层用滚动数组覆盖上一层，并将 $dp_k[n]$ 更新到答案。这样最终取到的是至少一个、至多 `m` 个子数组的最优值。

**代码**

```cpp
using LL = long long;
using ULL = unsigned long long;

const LL NEG = -(1LL << 62);

class Solution {
public:
    long long maximumSum(vector<int>& nums, int m, int l, int r) {
        int n = nums.size();

        vector<LL> pre(n + 1);
        for (int i = 1; i <= n; i++) {
            pre[i] = pre[i - 1] + nums[i - 1];
        }

        vector<LL> dp(n + 1, 0);
        vector<LL> ndp(n + 1, NEG);

        LL ans = NEG;
        int lim = min(m, n / l);

        for (int k = 1; k <= lim; k++) {
            fill(ndp.begin(), ndp.end(), NEG);

            deque<int> dq;

            for (int i = 1; i <= n; i++) {
                int j = i - l;

                if (j >= 0 && dp[j] != NEG) {
                    LL val = dp[j] - pre[j];

                    while (!dq.empty()) {
                        int x = dq.back();
                        if (dp[x] - pre[x] > val) {
                            break;
                        }
                        dq.pop_back();
                    }

                    dq.push_back(j);
                }

                while (!dq.empty() && dq.front() < i - r) {
                    dq.pop_front();
                }

                ndp[i] = ndp[i - 1];

                if (!dq.empty()) {
                    int x = dq.front();
                    ndp[i] = max(ndp[i], pre[i] + dp[x] - pre[x]);
                }
            }

            ans = max(ans, ndp[n]);
            dp.swap(ndp);
        }

        return ans;
    }
};
```

**复杂度分析**

设 $K=\min(m,\lfloor n/l\rfloor)$。

每一层 DP 中，每个下标最多进出单调队列各一次。

时间复杂度：$O(Kn)$，最坏为 $O(mn)$。

空间复杂度：$O(n)$。

**变形**

如果要求恰好选择 `m` 个子数组，只需返回第 `m` 层的 `dp[n]`，不再对所有层取最大值。如果允许选择零个子数组，则最终答案还要与 `0` 取最大值。若 `n` 和 `m` 同时扩大到 $10^5$，$O(mn)$ 无法通过，需要像下一题一样消去“选择数量”这一维。

## 3957 - Maximum Sum of M Non-Overlapping Subarrays II

**题意**

题目链接：[Maximum Sum of M Non-Overlapping Subarrays II](https://leetcode.com/problems/maximum-sum-of-m-non-overlapping-subarrays-ii/description/)

题意与上一题完全相同，但数据范围扩大为：

- `1 <= n <= 10^5`
- `-10^5 <= nums[i] <= 10^5`
- `1 <= m <= n`
- `1 <= l <= r <= n`

上一题的 $O(mn)$ 动态规划无法通过，需要消去选择数量这一维。

**解题思路**

### 1. WQS 二分的基本思想

WQS 二分也称 Alien Trick、带权二分或拉格朗日松弛。它适用于一类带有“恰好或至多选择 `m` 个对象”限制的最优化问题。

其直观含义是：暂时取消数量限制，但每选择一个对象都收取罚金 `x`。

- 罚金较小时，多选一个子数组的代价较低，最优方案倾向于选择更多子数组；
- 罚金较大时，多选一个子数组的代价较高，最优方案倾向于选择更少子数组；
- 二分罚金，就可以找到选择数量跨过 `m` 的临界位置。

设：

$$
F(k)=\text{恰好选择 }k\text{ 个子数组时的最大原始总和}
$$

给每个子数组收取罚金 `x` 后，需要计算：

$$
G(x)=\max_{k\ge 1}\{F(k)-kx\}
$$

固定 `x` 后，不再要求记录某个确定的 `k`，选择数量这一维便可以从 DP 中消失。不过状态仍需同时记录：

$$
(\text{扣除罚金后的价值},\text{选择数量})
$$

比较两个状态时，先比较价值；价值相同则选择数量更多的状态。这个平局规则决定了临界罚金处二分谓词的方向。

### 2. 为什么选择数量关于罚金单调

取两个罚金 $x_1<x_2$，对应最优方案分别选择 $c_1,c_2$ 个子数组。

根据两次最优性的定义：

$$
F(c_1)-x_1c_1\ge F(c_2)-x_1c_2
$$

$$
F(c_2)-x_2c_2\ge F(c_1)-x_2c_1
$$

两式相加：

$$
(x_2-x_1)(c_1-c_2)\ge 0
$$

因为 $x_2-x_1>0$，所以 $c_1\ge c_2$。罚金增加时，最优选择数量单调不增，因此可以二分。

### 3. 固定罚金下的线性 DP

定义 `dp[i] = (val, cnt)`，表示只考虑前 `i` 个元素、至少选择一个子数组时的最优状态。

若最后一个子数组为 `(j, i]`，则仍有：

$$
i-r\le j\le i-l
$$

最后一个子数组之前有两种情况：

1. 它是第一个子数组，基础状态为 $(-pre[j],0)$；
2. 前 `j` 个元素中已经选择过子数组，基础状态为 $(dp[j].val-pre[j],dp[j].cnt)$。

令 $key[j]$ 为以上两个状态中较优的一个。选择最后一个子数组后，需要加上：

$$
(pre[i]-x,1)
$$

因此：

$$
dp[i]=\max\left(
dp[i-1],
\max_{j\in[i-r,i-l]}key[j]+(pre[i]-x,1)
\right)
$$

窗口 `[i-r, i-l]` 继续使用单调队列维护最优 `key[j]`。一次 `run(x)` 的时间复杂度为 $O(n)$。

### 4. 临界罚金与答案恢复

先计算 `run(0)`：

- 如果无罚金最优方案的数量已经不超过 `m`，数量限制没有截断全局最优解，直接返回其原始价值；
- 如果数量大于 `m`，二分最大的整数罚金 `x`，使 `run(x).cnt >= m`。

本题中 $F(k)$ 具有离散凹性，即增加第 `k` 个子数组的边际收益随 `k` 增大而不增。该性质保证在临界罚金处，选择 `m` 个子数组也是罚金问题的最优方案之一。因此：

$$
G(x)=F(m)-mx
$$

将罚金加回即可恢复原始答案：

$$
F(m)=G(x)+mx
$$

代码中对应：

```cpp
z.val + lo * m
```

必须注意，WQS 二分并不是看到任意一次 `cnt >= m` 就立即恢复答案；二分谓词的作用是找到数量发生变化的临界罚金。只有结合 $F(k)$ 的离散凹性和平局规则，才能保证选择 `m` 个也是临界罚金下的最优方案。

**代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

#ifdef LOCAL
#include "debug.h"
#else
#define debug(...) 42
#endif
using LL = long long;
using ULL = unsigned long long;

const LL INF = (1LL << 62);

class Solution {
    struct Sta {
        LL val;
        int cnt;
    };

    // 判断状态 a 是否优于状态 b：先比较价值，价值相同时选择段数更多的状态。
    bool cmp(const Sta& a, const Sta& b) {
        if (a.val != b.val) {
            return a.val > b.val;
        }
        return a.cnt > b.cnt;
    }

    // 计算每选择一段扣除罚金 x 后的最大价值，以及平局时最多选择的段数。
    Sta run(const vector<LL>& ps, int n, int l, int r, LL x) {
        vector<Sta> dp(n + 1, {-INF, 0});
        vector<Sta> ky(n + 1, {-INF, 0});
        deque<int> q;

        for (int i = 1; i <= n; i++) {
            while (!q.empty() && q.front() < i - r) {
                q.pop_front();
            }

            int j = i - l;
            if (j >= 0) {
                Sta a = {-ps[j], 0};
                Sta b = {-INF, 0};

                if (dp[j].val > -INF / 2) {
                    b = {dp[j].val - ps[j], dp[j].cnt};
                }

                ky[j] = cmp(a, b) ? a : b;

                while (!q.empty() && !cmp(ky[q.back()], ky[j])) {
                    q.pop_back();
                }
                q.push_back(j);
            }

            dp[i] = dp[i - 1];

            if (!q.empty()) {
                j = q.front();

                Sta z = {
                    ky[j].val + ps[i] - x,
                    ky[j].cnt + 1
                };

                if (cmp(z, dp[i])) {
                    dp[i] = z;
                }
            }
        }

        return dp[n];
    }

public:
    long long maximumSum(vector<int>& nums, int m, int l, int r) {
        int n = nums.size();

        vector<LL> ps(n + 1);
        for (int i = 1; i <= n; i++) {
            ps[i] = ps[i - 1] + nums[i - 1];
        }

        Sta z = run(ps, n, l, r, 0);

        if (z.cnt <= m) {
            return z.val;
        }

        LL lo = 0;
        LL hi = 2LL * n * 100000 + 1;

        while (lo < hi) {
            LL mid = (lo + hi + 1) / 2;

            z = run(ps, n, l, r, mid);

            if (z.cnt >= m) {
                lo = mid;
            } else {
                hi = mid - 1;
            }
        }

        z = run(ps, n, l, r, lo);

        return z.val + lo * m;
    }
};
```

**复杂度分析**

设 $A=\max|nums[i]|$。罚金上界为 $O(nA)$，每次 `run(x)` 使用单调队列在线性时间内完成。

时间复杂度：$O(n\log(nA))$。本题中二分次数不超过约 `36` 次。

空间复杂度：$O(n)$。

**变形**

如果要求恰好选择 `m` 个子数组，可以直接使用 WQS 二分恢复 $F(m)$，不需要先判断 `run(0).cnt`。如果每选择一个子数组还需要额外支付固定成本 `c`，只需把总罚金改为 `x + c`。如果子数组长度固定为 `l`，每个右端点只对应 `j = i - l`，固定罚金下不再需要单调队列。

WQS 二分不是适用于所有数量限制问题。使用前必须确认：固定罚金后的问题能够更快求解，最优选择数量随罚金单调变化，并且目标函数关于选择数量具有能够在临界点恢复答案的凸性或凹性结构。
