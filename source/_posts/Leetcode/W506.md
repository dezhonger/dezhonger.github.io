## 3959 - Check Good Integer

**题意**

题目链接：[Check Good Integer](https://leetcode.com/problems/check-good-integer/description/)

给定一个正整数 `n`。设它的十进制数位为 $d_1,d_2,\ldots,d_k$，定义：

$$
digitSum=\sum_{i=1}^{k}d_i
$$

$$
squareSum=\sum_{i=1}^{k}d_i^2
$$

当且仅当

$$
squareSum-digitSum\ge 50
$$

时，称 `n` 为好整数。判断 `n` 是否为好整数。

- `1 <= n <= 10^9`

**解题思路**

直接按照定义处理每一位数字即可。

每次令 `d = n % 10` 取出最低位，把 `d` 加入数位和，把 `d * d` 加入平方和，再令 `n /= 10` 删除最低位。处理完所有数位后判断两者之差是否至少为 `50`。

**代码**

```cpp
class Solution {
public:
    bool checkGoodInteger(int n) {
        int digitSum = 0;
        int squareSum = 0;

        while (n > 0) {
            int d = n % 10;
            digitSum += d;
            squareSum += d * d;
            n /= 10;
        }

        return squareSum - digitSum >= 50;
    }
};
```

**复杂度分析**

设 `n` 的十进制位数为 $D$。

时间复杂度：$O(D)$。

空间复杂度：$O(1)$。

**变形**

如果需要判断一个区间内有多少个好整数，可以直接枚举区间；当区间扩大到 $10^{18}$ 级别时，则需要把 `digitSum` 和 `squareSum` 的差加入数位 DP 状态。

## 3960 - Frequency Balance Subarray

**题意**

题目链接：[Frequency Balance Subarray](https://leetcode.com/problems/frequency-balance-subarray/description/)

给定整数数组 `nums`，定义频率平衡子数组：

- 若子数组只包含一种不同的值，则它是频率平衡的；
- 否则，必须存在正整数 `f`，使每种不同的值都恰好出现 `f` 次或 `2f` 次，并且这两种频率都实际出现。

返回最长频率平衡子数组的长度。

- `1 <= nums.length <= 10^3`
- `1 <= nums[i] <= 10^9`

**解题思路**

### 1. 枚举左端点

数据范围只有 $n\le 1000$，可以枚举左端点 `l`，再向右扩展 `r`。对于固定的 `l`，每加入一个 `nums[r]`，只会让一种数的出现次数从 `c` 变成 `c + 1`。

如果能够在 $O(1)$ 时间内更新并判断当前频率分布，总时间复杂度就是 $O(n^2)$。

先对 `nums` 离散化，并维护：

- `cnt[x]`：离散值 `x` 在当前子数组中出现了多少次；
- `frc[f]`：当前有多少种值恰好出现 `f` 次；
- `typ`：有多少个正频率 `f` 满足 `frc[f] > 0`；
- `pairCnt`：当前有多少对正频率 `(f, 2f)` 同时存在。

### 2. 如何在常数时间内维护频率桶

当某个数的频率从 `c` 变成 `c + 1` 时，只需执行：

```cpp
frc[c]--;
frc[c + 1]++;
```

一个频率桶只有在 `frc[f]` 从 `0` 变成正数，或从正数变成 `0` 时，才会改变 `typ`。

频率 `f` 只可能参与两种二倍关系：

- 它作为较小频率，与 `2f` 组成 `(f, 2f)`；
- 当 `f` 为偶数时，它作为较大频率，与 `f / 2` 组成 `(f / 2, f)`。

因此在切换频率桶 `f` 的活跃状态前后，只需检查这两个关系，就能在 $O(1)$ 时间内维护 `pairCnt`。

### 3. 合法条件

当前子数组合法当且仅当：

- `distinct == 1`，即只包含一种不同的值；或
- `typ == 2 && pairCnt > 0`。

第二个条件表示当前恰好存在两种正频率，并且它们构成 `(f, 2f)`。由于两个频率桶都非空，也就保证了题目要求的两种频率都实际出现。

需要特别注意：若多个不同值都只出现一次，此时只有一种频率 `1`，但子数组包含多种值，因此并不合法。

**代码**

```cpp
class Solution {
public:
    int maxBalancedSubarray(vector<int>& nums) {
        int n = nums.size();

        vector<int> xs = nums;
        sort(xs.begin(), xs.end());
        xs.erase(unique(xs.begin(), xs.end()), xs.end());

        int m = xs.size();
        vector<int> id(n);

        for (int i = 0; i < n; i++) {
            id[i] = lower_bound(xs.begin(), xs.end(), nums[i]) - xs.begin();
        }

        int ans = 1;

        for (int l = 0; l < n; l++) {
            vector<int> cnt(m, 0);
            vector<int> frc(n + 1, 0);

            int distinct = 0;
            int typ = 0;
            int pairCnt = 0;

            // 把频率桶 f 中的值种类数增加 delta。
            auto change = [&](int f, int delta) {
                if (f == 0) {
                    return;
                }

                bool oldActive = frc[f] > 0;

                // 先删除旧状态中与 f 有关的二倍关系。
                if (oldActive) {
                    if (2 * f <= n && frc[2 * f] > 0) {
                        pairCnt--;
                    }
                    if (f % 2 == 0 && frc[f / 2] > 0) {
                        pairCnt--;
                    }
                }

                frc[f] += delta;
                bool newActive = frc[f] > 0;

                if (oldActive != newActive) {
                    typ += newActive ? 1 : -1;
                }

                // 再加入新状态中与 f 有关的二倍关系。
                if (newActive) {
                    if (2 * f <= n && frc[2 * f] > 0) {
                        pairCnt++;
                    }
                    if (f % 2 == 0 && frc[f / 2] > 0) {
                        pairCnt++;
                    }
                }
            };

            for (int r = l; r < n; r++) {
                int x = id[r];
                int c = cnt[x];

                if (c == 0) {
                    distinct++;
                }

                change(c, -1);
                cnt[x]++;
                change(c + 1, 1);

                if (distinct == 1 || (typ == 2 && pairCnt > 0)) {
                    ans = max(ans, r - l + 1);
                }
            }
        }

        return ans;
    }
};
```

**复杂度分析**

离散化需要 $O(n\log n)$。共有 $O(n^2)$ 个候选子数组，每次扩展右端点只进行常数次频率桶更新。

总时间复杂度：$O(n^2)$。

空间复杂度：$O(n)$。

**变形**

如果合法频率可以是 `f`、`2f`、`4f` 三种，只需把活跃频率之间的关系扩展成二倍链，并要求恰好存在三个活跃桶。若合法频率之间没有固定的倍数关系，通常需要使用有序集合维护当前活跃频率，复杂度会增加到 $O(n^2\log n)$。

## 3961 - Maximize Sum of Device Ratings

**题意**

题目链接：[Maximize Sum of Device Ratings](https://leetcode.com/problems/maximize-sum-of-device-ratings/description/)

给定 `m` 台设备，每台设备恰好有 `n` 个单元，`units[i][j]` 表示第 `i` 台设备第 `j` 个单元的容量。一台非空设备的评分是其中所有单元容量的最小值，空设备的评分为 `0`。

可以进行任意次以下操作：

- 选择一台以前没有作为源设备使用过的设备 `i`；
- 从中删除恰好一个单元，并把它加入另一台设备；
- 将 `i` 标记为已经使用，之后不能再次作为源设备。

求所有设备评分之和的最大值。

- `1 <= m == units.length <= 10^5`
- `1 <= n == units[i].length <= 10^5`
- `m * n <= 2 * 10^5`
- `1 <= units[i][j] <= 10^5`

**解题思路**

### 1. 每台设备只需关注最小值和次小值

设第 `i` 台设备的最小容量和次小容量分别为：

$$
a_i\le b_i
$$

若希望提高这台设备的评分，唯一有意义的操作就是移走最小单元 $a_i$。移走后，这台设备的评分变成 $b_i$；移走其他单元不会改变最小值，也就不会提高评分。

### 2. 所有被移走的最小值都送到同一台设备

接收最小单元的设备评分可能被降低。把所有被移走的 $a_i$ 都送到同一台设备，可以把损失集中在一个汇点设备上，其余设备都取得次小值评分。

汇点设备的最终评分一定是所有单元中的全局最小值：

$$
g=\min_i a_i
$$

若暂时让每台设备都贡献次小值，得到：

$$
\sum_i b_i
$$

选择设备 `p` 作为汇点后，它不再贡献 $b_p$，而是贡献 $g$，所以答案为：

$$
\sum_i b_i-b_p+g
$$

为了最大化结果，应选择次小值最小的设备作为汇点：

$$
\boxed{\sum_i b_i-\min_i b_i+\min_i a_i}
$$

若每台设备只有一个单元，任何转移都会让源设备变空，并且不能产生更大的评分和，最优方案是不进行操作，直接返回所有单元之和。

**代码**

```cpp
class Solution {
public:
    long long maxRatings(vector<vector<int>>& units) {
        int m = units.size();
        int n = units[0].size();

        if (n == 1) {
            long long ans = 0;
            for (auto& row : units) {
                ans += row[0];
            }
            return ans;
        }

        long long sumSecond = 0;
        int globalMin = INT_MAX;
        int minSecond = INT_MAX;

        for (auto& row : units) {
            int mn1 = INT_MAX;
            int mn2 = INT_MAX;

            for (int x : row) {
                if (x < mn1) {
                    mn2 = mn1;
                    mn1 = x;
                } else if (x < mn2) {
                    mn2 = x;
                }
            }

            globalMin = min(globalMin, mn1);
            minSecond = min(minSecond, mn2);
            sumSecond += mn2;
        }

        return sumSecond - minSecond + globalMin;
    }
};
```

**复杂度分析**

设所有设备的单元总数为 $S=m\times n$。

时间复杂度：$O(S)$，每个单元只扫描一次。

空间复杂度：$O(1)$，不计输入数组。

**变形**

如果每台设备可以移走至多 `k` 个单元，则每台设备需要关注前 `k + 1` 小的容量，问题会变成如何在多个设备之间分配“提升层数”和承受被转移小值的汇点，通常需要动态规划或边际收益排序。

## 3962 - Maximum Subarray Sum After at Most K Swaps

**题意**

题目链接：[Maximum Subarray Sum After at Most K Swaps](https://leetcode.com/problems/maximum-subarray-sum-after-at-most-k-swaps/description/)

给定整数数组 `nums` 和整数 `k`。最多进行 `k` 次交换，每次可以选择任意两个下标并交换对应元素。返回操作后能够得到的最大非空子数组和。

- `1 <= nums.length <= 1500`
- `-10^5 <= nums[i] <= 10^5`
- `0 <= k <= nums.length`

**解题思路**

### 1. 固定目标子数组

枚举最终要求和的子数组 `[l, r]`。设它的原始元素和为 `cur`，长度为 `len`，区间外有 `os = n - len` 个元素。

交换两个区间内部元素不会改变区间和，交换两个区间外部元素同样不会改变区间和。真正有意义的交换一定发生在区间内部与区间外部之间。

设区间内部元素从小到大排列为：

$$
A_1\le A_2\le\cdots
$$

区间外部元素从大到小排列为：

$$
B_1\ge B_2\ge\cdots
$$

固定交换次数 `t` 时，最优方案一定是移走区间内最小的 `t` 个元素，并移入区间外最大的 `t` 个元素。交换后的区间和为：

$$
cur+\sum_{i=1}^{t}(B_i-A_i)
$$

交换次数不能超过：

$$
lim=\min(k,len,os)
$$

### 2. 有收益的交换构成前缀

由于 $A_i$ 单调不减、$B_i$ 单调不增，所以边际收益：

$$
B_i-A_i
$$

单调不增。所有严格为正的交换收益必然构成一个前缀。

因此最优交换次数 `c` 是最大的满足以下条件的整数：

$$
0\le c\le lim
$$

$$
A_c<B_c
$$

若下一对仍满足 $A_{c+1}<B_{c+1}$，就应该继续交换；若当前最后一对满足 $A_c\ge B_c$，就应该减少一次交换。

### 3. 右端点扩展时，最优交换次数最多变化 1

固定左端点 `l`，让右端点 `r` 逐步向右扩展。每次只发生一件事：一个元素从区间外集合移动到区间内集合。

把“内部元素小于外部元素”看成一个二分图匹配条件，有收益的交换数量就是最大可匹配对数。移动右端点时：

- 从外部集合删除一个元素，最大匹配数最多减少 `1`；
- 向内部集合加入一个元素，最大匹配数最多增加 `1`。

合并来看，相邻两个右端点对应的最优交换次数最多只变化 `1`。限制 `lim` 本身也最多变化 `1`。

所以不需要对每个子数组二分交换次数。沿用上一个右端点的 `c` 后，只需：

1. 先令 `c = min(c, lim)`；
2. 检查能否增加到 `c + 1`；
3. 否则检查是否需要减少到 `c - 1`。

这样每个子数组只需常数次顺序统计查询。

### 4. 两棵树状数组维护顺序统计

对 `nums` 离散化，分别用两棵树状数组维护：

- `in`：当前子数组内的元素；
- `out`：当前子数组外的元素。

每棵树状数组同时维护每个值的出现次数和元素总和，从而支持：

- `kth(k)`：集合中第 `k` 小的值；
- `sumSmallest(k)`：集合中最小的 `k` 个元素之和。

区间外最大的 `c` 个元素之和等于：

$$
out.total-out.sumSmallest(os-c)
$$

区间内最小的 `c` 个元素之和为：

$$
in.sumSmallest(c)
$$

因此当前子数组能够取得的最大和是：

$$
\boxed{
cur+left(out.total-out.sumSmallest(os-c)\right)-in.sumSmallest(c)
}
$$

**代码**

```cpp
using LL = long long;

class Solution {
    struct BIT {
        int n;
        int num;
        LL total;
        vector<int> xs;
        vector<int> cnt;
        vector<LL> sum;

        BIT(const vector<int>& values)
            : n(values.size()), num(0), total(0), xs(values),
              cnt(n + 1, 0), sum(n + 1, 0) {}

        void add(int p, int delta) {
            num += delta;
            total += 1LL * xs[p - 1] * delta;

            for (int i = p; i <= n; i += i & -i) {
                cnt[i] += delta;
                sum[i] += 1LL * xs[p - 1] * delta;
            }
        }

        int queryCount(int p) const {
            int ans = 0;
            for (int i = p; i > 0; i -= i & -i) {
                ans += cnt[i];
            }
            return ans;
        }

        LL querySum(int p) const {
            LL ans = 0;
            for (int i = p; i > 0; i -= i & -i) {
                ans += sum[i];
            }
            return ans;
        }

        // 返回第 k 小元素所在的离散化下标，结果为 1-based。
        int kthIndex(int k) const {
            int p = 0;
            int acc = 0;

            int step = 1;
            while ((step << 1) <= n) {
                step <<= 1;
            }

            for (; step > 0; step >>= 1) {
                int np = p + step;
                if (np <= n && acc + cnt[np] < k) {
                    p = np;
                    acc += cnt[np];
                }
            }

            return p + 1;
        }

        int kth(int k) const {
            return xs[kthIndex(k) - 1];
        }

        LL sumSmallest(int k) const {
            if (k == 0) {
                return 0;
            }

            int p = kthIndex(k);
            int before = queryCount(p - 1);
            LL ans = querySum(p - 1);

            ans += 1LL * (k - before) * xs[p - 1];
            return ans;
        }
    };

public:
    long long maxSum(vector<int>& nums, int k) {
        int n = nums.size();

        vector<int> xs = nums;
        sort(xs.begin(), xs.end());
        xs.erase(unique(xs.begin(), xs.end()), xs.end());

        vector<int> id(n);
        for (int i = 0; i < n; i++) {
            id[i] = lower_bound(xs.begin(), xs.end(), nums[i]) - xs.begin() + 1;
        }

        LL ans = LLONG_MIN;

        for (int l = 0; l < n; l++) {
            BIT in(xs);
            BIT out(xs);

            for (int p : id) {
                out.add(p, 1);
            }

            LL cur = 0;
            int c = 0;

            for (int r = l; r < n; r++) {
                out.add(id[r], -1);
                in.add(id[r], 1);
                cur += nums[r];

                int len = r - l + 1;
                int os = n - len;
                int lim = min({k, len, os});

                c = min(c, lim);

                // 尝试加入第 c + 1 个有正收益的交换。
                if (c < lim && in.kth(c + 1) < out.kth(os - c)) {
                    c++;
                } else if (c > 0 && in.kth(c) >= out.kth(os - c + 1)) {
                    // 当前最后一个交换已经没有正收益。
                    c--;
                }

                LL best = cur;
                best += out.total - out.sumSmallest(os - c);
                best -= in.sumSmallest(c);

                ans = max(ans, best);
            }
        }

        return ans;
    }
};
```

**正确性说明**

固定子数组后，任意有效操作都可以只保留区间内外之间的交换，因为区间内部或区间外部的交换不会改变目标子数组的元素集合。

固定交换次数 `t` 时，若移出的内部元素不是最小的 `t` 个之一，把它替换成更小的内部元素不会使结果变差；若移入的外部元素不是最大的 `t` 个之一，把它替换成更大的外部元素同样不会使结果变差。因此最优收益一定是前 `t` 对 `(B_i-A_i)` 的和。

边际收益单调不增，所以所有有正收益的交换构成前缀。代码维护的 `c` 正是该前缀长度与 `lim` 的较小值，并利用相邻右端点之间最多变化 `1` 的性质完成正确更新。

算法枚举了所有非空子数组，并为每个子数组计算其最多 `k` 次交换后的最优和，因此所有候选值中的最大值就是全局答案。

**复杂度分析**

共有 $O(n^2)$ 个子数组。移动右端点、调整交换次数和计算答案只需要常数次树状数组操作，每次操作为 $O(\log n)$。

时间复杂度：$O(n^2\log n)$。

空间复杂度：$O(n)$。

**变形**

如果只要求固定子数组在至多 `k` 次交换后的最大和，可以直接维护两个有序集合，或对交换次数二分。若交换只能发生在相邻位置，代价将与元素移动距离有关，最优方案不再只是比较区间内外的元素大小，通常需要最小费用流、动态规划或带距离代价的匹配方法。
