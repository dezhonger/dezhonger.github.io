## 题意

定义

$$
g(m,n) = (m \oplus n) + (m \vee n) + (m \wedge n)
$$

其中 $\oplus,\vee,\wedge$ 分别表示按位异或、按位或、按位与。

再定义

$$
G(N) = \sum_{n=0}^{N}\sum_{k=0}^{n} g(k,n-k)
$$

已知：

- $G(10)=754$
- $G(10^2)=583766$

要求计算：

$$
G(10^{18}) \bmod (10^9+7)
$$

## 解题思路

### 第一步：先化简 $g(m,n)$

我们先看：

$$
m \vee n = (m \oplus n) + (m \wedge n)
$$

这是一个很经典的按位恒等式。

所以：

$$
g(m,n)=(m\oplus n)+(m\vee n)+(m\wedge n)=2(m\vee n)
$$

于是原式变成：

$$
G(N)=2\sum_{n=0}^{N}\sum_{k=0}^{n}(k\vee(n-k))
$$

把内外层一起换个写法。令 $a=k,b=n-k$，那么 $a,b\ge 0$ 且 $a+b\le N$，于是

$$
G(N)=2\sum_{a+b\le N}(a\vee b)
$$

设

$$
T(N)=\sum_{a+b\le N}(a\vee b)
$$

那么最终要求的就是

$$
G(N)=2T(N)
$$

所以问题已经转化成：

> 求所有满足 $a,b\ge 0,\ a+b\le N$ 的数对中，$a\vee b$ 的总和。

---

### 第二步：按二进制最低位拆分

这是本题的核心。

把 $a,b$ 写成：

$$
a=2x+u,\qquad b=2y+v,\qquad u,v\in\{0,1\}
$$

那么有：

$$
a\vee b = 2(x\vee y) + (u\vee v)
$$

原因很简单：

- 高位整体右移掉最低位之后，答案会整体乘 $2$
- 最低位只由 $u\vee v$ 决定

所以我们就可以按照 $u,v$ 分类讨论，把大问题递归成小问题。

---

### 第三步：讨论 $N$ 的奇偶性

#### 情况一：$N=2m$

此时约束为：

$$
2x+u+2y+v\le 2m
$$

也就是

$$
2(x+y)+(u+v)\le 2m
$$

分类讨论：

- 当 $(u,v)=(0,0)$ 时，$x+y\le m$
- 当 $(u,v)=(0,1),(1,0),(1,1)$ 时，$x+y\le m-1$

因此：

$$
\begin{aligned}
T(2m)
&= \sum_{x+y\le m} 2(x\vee y) \\
&\quad + 3\sum_{x+y\le m-1}\bigl(2(x\vee y)+1\bigr)
\end{aligned}
$$

设

$$
C(n)=\#\{(x,y)\mid x\ge 0,\ y\ge 0,\ x+y\le n\}
$$

那么

$$
C(n)=\frac{(n+1)(n+2)}{2}
$$

于是上式整理得到：

$$
T(2m)=2T(m)+6T(m-1)+3C(m-1)
$$

---

#### 情况二：$N=2m+1$

此时约束为：

$$
2(x+y)+(u+v)\le 2m+1
$$

分类讨论：

- 当 $(u,v)=(0,0),(0,1),(1,0)$ 时，$x+y\le m$
- 当 $(u,v)=(1,1)$ 时，$x+y\le m-1$

于是：

$$
\begin{aligned}
T(2m+1)
&= \sum_{x+y\le m} 2(x\vee y) \\
&\quad + 2\sum_{x+y\le m}\bigl(2(x\vee y)+1\bigr) \\
&\quad + \sum_{x+y\le m-1}\bigl(2(x\vee y)+1\bigr)
\end{aligned}
$$

整理得到：

$$
T(2m+1)=6T(m)+2T(m-1)+2C(m)+C(m-1)
$$

---

### 第四步：边界条件

显然：

$$
T(-1)=0,\qquad T(0)=0
$$

因为当 $N=0$ 时，唯一的数对是 $(0,0)$，并且

$$
0\vee 0=0
$$

## 代码

```cpp
#include <bits/stdc++.h>
using namespace std;

#ifdef LOCAL
#include "debug.h"
#else
#define debug(...) 42
#endif
using LL = long long;
using ULL = unsigned long long;

const LL mod = 1000000007LL;
const LL iv2 = (mod + 1) / 2;

map<LL, LL> mp;

/* 计算 C(n) = 满足 x>=0, y>=0, x+y<=n 的数对个数 */
LL c2(LL n) {
    if (n < 0) return 0;
    LL a = (n + 1) % mod;
    LL b = (n + 2) % mod;
    return a * b % mod * iv2 % mod;
}

/* 计算 T(n) = sum_{a+b<=n} (a|b) */
LL dfs(LL n) {
    if (n <= 0) return 0;
    if (mp.count(n)) return mp[n];

    LL m = n >> 1;
    LL ans = 0;

    if ((n & 1) == 0) {
        ans = (2 * dfs(m) + 6 * dfs(m - 1) + 3 * c2(m - 1)) % mod;
    } else {
        ans = (6 * dfs(m) + 2 * dfs(m - 1) + 2 * c2(m) + c2(m - 1)) % mod;
    }

    return mp[n] = ans;
}

void solve() {
    LL n = 1000000000000000000LL;
    LL ans = 2 * dfs(n) % mod;
    cout << ans << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    auto st = chrono::steady_clock::now();

    solve();

    auto ed = chrono::steady_clock::now();
    double ms = chrono::duration<double, milli>(ed - st).count();
    cerr << "time: " << ms << " ms\n";

    return 0;
}
```

## 复杂度分析

设输入规模为 $N$。

时间复杂度：

$$
O(\log N)
$$

空间复杂度：

$$
O(\log N)
$$

其中空间主要来自记忆化状态和递归栈。

## 题目可以如何变形和扩展

### 变形 1：改成恰好 $a+b=N$

即求：

$$
\sum_{k=0}^{N} g(k,N-k)
$$

这个会比前缀和型更简单，因为不需要累计整个三角区域。

### 变形 2：改成只求一种按位运算

例如：

$$
\sum_{a+b\le N}(a\oplus b)
$$

或者：

$$
\sum_{a+b\le N}(a\wedge b)
$$

这类题同样可以按最低位拆分，写出类似递推。

### 变形 3：推广到三个数

比如：

$$
\sum_{a+b+c\le N}(a\vee b\vee c)
$$

思路还是类似的，但最低位分类会更多，推导会复杂很多。
