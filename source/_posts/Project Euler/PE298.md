## 题目描述

Larry 和 Robin 玩一个记忆游戏。随机从 $1\sim10$ 中等概率选取一个数字并报出，双方各自最多能记住 $5$ 个数字。当报出的数字在某人的记忆中时，该玩家得 $1$ 分；否则将该数字加入记忆（若已满则按各自的策略删除一个数字）。

两人的删除策略不同：

- **Larry**：删除“最久没有被叫到”的数字（LRU）。
- **Robin**：删除“在记忆中待得最久”的数字（FIFO）。

初始时两人记忆均为空。设 $50$ 轮后 Larry 的分数为 $L$，Robin 的分数为 $R$，求 $\mathbb E(|L-R|)$，保留八位小数。

## 解题思路

### 2.1 状态定义

游戏无后效性——未来得分和删除操作只取决于两人当前的记忆顺序和分数差。设分数差 $d = L - R$。

定义状态 $S = (A, B)$：

- $A$：Larry 的记忆序列，顺序为从最久没被叫到到最近被叫到。
- $B$：Robin 的记忆序列，顺序为从最早加入到最晚加入。

记 $P_t(S, d)$ 为第 $t$ 轮结束后处于状态 $S$ 且分数差为 $d$ 的概率。

### 2.2 最终答案的式子

最终答案为：

$$
\sum_S \sum_d |d| \cdot P_{50}(S, d)
$$

含义如下：枚举所有可能的最终状态 $S$ 和所有可能的分数差 $d$，将 $|d|$（即 $|L-R|$）乘以该情况发生的概率 $P_{50}(S, d)$，累加后得到期望值。

这正是期望的定义 $\mathbb E(X) = \sum_x x \cdot P(X=x)$，其中随机变量 $X = |L-R|$。

例如，若经过 50 轮后只可能出现三种情况：

| 情况 | 分数差 $d$ | 概率 $P$ |
|------|-----------|----------|
| 1    | $d = 1$   | $0.2$    |
| 2    | $d = -2$  | $0.5$    |
| 3    | $d = 0$   | $0.3$    |

则期望为：

$$
0.2 \times |1| + 0.5 \times |-2| + 0.3 \times |0| = 0.2 + 1.0 + 0 = 1.2
$$

注意 $|d|$ 取绝对值，因此 $d = +2$ 和 $d = -2$ 对答案的贡献相同。

### 2.3 转移规则

**Larry 的记忆序列 $A$：**

- 顺序规则：最左边是"最久没有被叫到"、最可能被淘汰的数字；最右边是"最近刚被叫到"或"最近才加入"的数字。
- 淘汰规则：当需要淘汰时，删除**最左边**的元素。
- 命中规则：命中后，将该数字移到**最右边**（更新最近使用时间）。

**Robin 的记忆序列 $B$：**

- 顺序规则：最左边是"最早加入记忆"、最可能被淘汰的数字；最右边是"最晚加入"的数字。
- 淘汰规则：当需要淘汰时，删除**最左边**的元素。
- 命中规则：命中后，**顺序不变**（因为 FIFO 只关心加入时间，不关心最近是否被叫到）。

**完整的更新过程（设当前状态为 $(A, B)$，本轮叫出数字 $x$）：**

1. **Larry 侧更新**：
   - 若 $x \in A$：得分 $+1$，将 $x$ 从序列中删去并加到序列最右边。
   - 若 $x \notin A$：若 $|A| = 5$（已满），删去最左边的元素；将 $x$ 加到序列最右边。

2. **Robin 侧更新**：
   - 若 $x \in B$：得分 $+1$，序列 $B$ 保持不变。
   - 若 $x \notin B$：若 $|B| = 5$（已满），删去最左边的元素；将 $x$ 加到序列最右边。

3. **分数差变化**：

$$
dd = [x \in A] - [x \in B]
$$

即 Larry 命中则 $dd + 1$，Robin 命中则 $dd - 1$。

### 2.4 状态规范化与压缩

**为什么要压缩状态？**

直接以数字 $1\sim10$ 作为标签，状态数会极其庞大。例如 Larry 记忆 $[1,2,3]$、Robin 记忆 $[4,5,6]$ 和 Larry 记忆 $[7,8,9]$、Robin 记忆 $[10,11,12]$（假设更大数字范围）表面上是不同的状态，但只要两人记忆中的相对顺序关系相同，游戏未来的发展完全一致——因为随机叫数是均匀分布，数字的名字本身没有意义。

将两个序列中第一次出现的数字重新编号为 $0,1,2,\dots$，即可将所有本质相同的状态合并。例如：

```
Larry: [7, 3, 9]   →  [0, 1, 2]
Robin: [3, 5, 7]   →  [1, 3, 0]
```

规范化后实际可达状态仅有数百个，相比于直接枚举 $10^5$ 量级的原始状态，极大提升了计算效率。

### 2.5 转移合并

**并集内 $c$ 个数字必须逐个枚举。**

设当前两人记忆的并集中有 $c$ 个不同数字。对于每个已在并集中的数字 $x$：

- $x$ 在 Larry 记忆中是否命中，决定 $dd$ 是否 $+1$
- $x$ 在 Robin 记忆中是否命中，决定 $dd$ 是否 $-1$
- 命中时 Larry 要将 $x$ 移到末尾（LRU 更新），Robin 命中后顺序不变（FIFO 特性）
- 未命中时两人都要将 $x$ 加入记忆

这些信息完全由"标签为 $x$ 的数字当前在谁的记忆中"决定。不同标签的并集内数字，其归属关系不同，导致转移后的状态也不同，必须逐个处理。每个的概率为 $\frac{1}{10}$（原题中等概率叫出 $1\sim10$ 中的某个数）。

**并集外 $10-c$ 个数字可以统一成一条转移。**

当前不在两人记忆中的数字，对当前状态的作用完全相同：

- Larry 一定不命中，Robin 一定不命中
- 两人都将其加入记忆（若已满各自按策略删除）
- 分数差变化 $dd = 0$

无论这些数字具体叫什么名字，转移结果都一样。因此可以将这 $10-c$ 个数字合并为一条"新数字"转移，概率为 $\frac{10-c}{10}$。

两者的区别在于：并集内数字会导致不同的分差变化和不同的后继状态（因为它们在双方记忆中的归属不同），而并集外数字无论具体是哪个，效果完全一致。

### 2.6 DP 递推

设 $dp[\text{state}][i]$ 为当前处于某状态、分差下标为 $i$（对应真实分差 $i - \text{off}$）的概率。

$$
dp_{t+1}(\text{state}', i + dd) \mathrel{+}= dp_t(\text{state}, i) \times p
$$

枚举 $t = 0\sim 49$ 轮后，将所有状态的 $|d|$ 加权求和即得期望。

## 代码

```cpp
#include <bits/stdc++.h>
using namespace std;

#ifdef LOCAL
#include "debug.h"
#else
#define debug(...) 42
#endif
using LL = long long;
using ULL = unsigned long long;
using LD = long double;

// ---------- 全局常数 ----------
const int num = 10;   // 数字种类数：1~10 共 10 个数字
const int cap = 5;    // 记忆容量：每人最多记住 5 个数字
const int turn = 50;  // 总轮数
const int off = turn; // 分差偏移量，用于将 [-turn, turn] 映射到 [0, 2*turn]
const int wid = 2 * turn + 1; // 分差数组长度

// ---------- 数据结构 ----------
// State: 游戏状态，包含 Larry 和 Robin 的记忆序列
struct State {
    vector<int> a; // Larry 的记忆序列（LRU 顺序）：最左=最久没被叫到，最右=最近被叫到
    vector<int> b; // Robin 的记忆序列（FIFO 顺序）：最左=最早加入，最右=最晚加入
};

// Transition: 状态转移记录
struct Transition {
    ULL key;        // 转移后的新状态编码
    int delta_diff; // 本轮对分差 d = L - R 的影响（+1/-1/0）
    LD p;           // 这条转移发生的概率
};

// encode: 将 (Larry序列, Robin序列) 编码为 ULL 整数键
//
// 编码结构（从高位到低位）：
//   [Larry长度] [Larry五个位置(用x+1表示，空位为0)] [Robin长度] [Robin五个位置]
// 每位用 11 进制存储（0~10），因此 cap=5 时刚好不冲突
ULL encode(const vector<int>& a, const vector<int>& b) {
    ULL v = a.size(); // 先存 Larry 序列长度
    for (int i = 0; i < cap; i++) {
        int x = 0;
        if (i < (int)a.size()) x = a[i] + 1; // 实际标签 x 存成 x+1，空位为 0
        v = v * 11 + x;
    }
    v = v * 6 + b.size(); // 存 Robin 序列长度（长度最多 5，用 6 进制足够）
    for (int i = 0; i < cap; i++) {
        int x = 0;
        if (i < (int)b.size()) x = b[i] + 1;
        v = v * 11 + x;
    }
    return v;
}

// decode: 将 ULL 整数键还原为 State 结构（encode 的逆过程）
State decode(ULL v) {
    // 先反向取出 Robin 的 5 个位置（11 进制）
    vector<int> rb(cap);
    for (int i = cap - 1; i >= 0; i--) {
        rb[i] = v % 11;
        v /= 11;
    }
    // 取出 Robin 长度
    int lb = v % 6;
    v /= 6;
    // 再反向取出 Larry 的 5 个位置
    vector<int> ra(cap);
    for (int i = cap - 1; i >= 0; i--) {
        ra[i] = v % 11;
        v /= 11;
    }
    // 最后剩下 Larry 长度
    int la = v;
    State s;
    // 解码时减 1 还原原始标签
    for (int i = 0; i < la; i++) s.a.push_back(ra[i] - 1);
    for (int i = 0; i < lb; i++) s.b.push_back(rb[i] - 1);
    return s;
}

// canonicalize: 将状态规范化（状态压缩的核心）
//
// 规则：按 Larry序列、Robin序列中第一次出现的顺序，将数字重新编号为 0,1,2,...
// 这样本质相同但具体数字不同的状态会变成同一个表示
ULL canonicalize(const vector<int>& a, const vector<int>& b) {
    vector<int> id(num + 1, -1); // id[x] = 原标签 x 被重新编号后的值，未出现则为 -1
    int cnt = 0;                // 当前已分配的新编号数量
    vector<int> na;

    // 先扫描 Larry 的序列分配新编号
    for (int x : a) {
        if (id[x] == -1) id[x] = cnt++;
        na.push_back(id[x]);
    }
    // 再扫描 Robin 的序列（已出现的使用相同编号）
    vector<int> nb;
    for (int x : b) {
        if (id[x] == -1) id[x] = cnt++;
        nb.push_back(id[x]);
    }
    // 规范化后编码
    return encode(na, nb);
}

// go: 给定当前状态和被叫出的数字 x，计算下一状态和分差变化
//
// 参数：
//   a: Larry 当前记忆序列
//   b: Robin 当前记忆序列
//   x: 本轮叫出的数字标签
// 返回值：
//   first:  下一状态的编码
//   second: 本轮分差变化 delta_diff
pair<ULL, int> go(const vector<int>& a, const vector<int>& b, int x) {
    int delta_diff = 0;

    // ---- Larry 侧更新（LRU 策略）----
    vector<int> na = a;
    auto ia = find(na.begin(), na.end(), x);
    if (ia != na.end()) {
        // Larry 命中：得分 +1，将 x 移到序列最右边（更新最近使用时间）
        delta_diff++;
        na.erase(ia);
        na.push_back(x);
    } else {
        // Larry 未命中：若已满，删掉最左边的元素（最久没被叫到的）
        if ((int)na.size() == cap) na.erase(na.begin());
        na.push_back(x); // 新数字加入序列最右边
    }

    // ---- Robin 侧更新（FIFO 策略）----
    vector<int> nb = b;
    auto ib = find(nb.begin(), nb.end(), x);
    if (ib != nb.end()) {
        // Robin 命中：得分 +1，但 FIFO 顺序不变（只关心加入时间）
        delta_diff--;
    } else {
        // Robin 未命中：若已满，删掉最左边的元素（最早加入的）
        if ((int)nb.size() == cap) nb.erase(nb.begin());
        nb.push_back(x);
    }

    // 更新后需规范化，合并本质相同的状态
    ULL nk = canonicalize(na, nb);
    return {nk, delta_diff};
}

// getTransitions: 生成某个状态的所有可能转移（记忆化）
//
// 当前状态下，下一轮叫出的数字分为两类：
//   1. 已在两人记忆并集中的数字（逐个枚举，每个概率 1/num）
//   2. 不在并集中的数字（合并成一条转移，概率 (num-cnt)/num）
const vector<Transition>& getTransitions(ULL key, unordered_map<ULL, vector<Transition>>& mem) {
    auto it = mem.find(key);
    if (it != mem.end()) return it->second; // 已计算过，直接返回

    State s = decode(key);

    // has[x] = 1 表示标签 x 出现在当前两人记忆的并集中
    vector<int> has(num, 0);
    int cnt = 0;
    for (int x : s.a) {
        if (!has[x]) { has[x] = 1; cnt++; }
    }
    for (int x : s.b) {
        if (!has[x]) { has[x] = 1; cnt++; }
    }

    vector<Transition> res;

    // ---- 第一类：叫到并集中的数字 ----
    for (int x = 0; x < num; x++) {
        if (has[x]) {
            auto [nk, dd] = go(s.a, s.b, x);
            res.push_back({nk, dd, 1.0L / num}); // 每个概率 1/num
        }
    }

    // ---- 第二类：叫到并集外的"新数字" ----
    if (cnt < num) {
        // 用 cnt 作为"新数字"的临时标签
        auto [nk, dd] = go(s.a, s.b, cnt);
        res.push_back({nk, dd, (LD)(num - cnt) / num});
    }

    auto ret = mem.emplace(key, move(res));
    return ret.first->second;
}

// solve: 主函数，执行 50 轮概率 DP 并输出答案
void solve() {
    auto beg = chrono::steady_clock::now();

    // dp[state][i] = 当前处于 state，分差下标为 i（对应真实分差 i-off）的概率
    unordered_map<ULL, vector<LD>> dp, ndp;
    // mem 记忆化每个状态的转移表
    unordered_map<ULL, vector<Transition>> mem;

    // 初始状态：两人记忆均为空，分差 d=0，概率为 1
    ULL ini = encode({}, {});
    dp[ini] = vector<LD>(wid, 0);
    dp[ini][off] = 1;

    // ---- 50 轮概率 DP ----
    for (int t = 0; t < turn; t++) {
        ndp.clear();
        for (auto& [key, vec] : dp) {
            const vector<Transition>& trs = getTransitions(key, mem);
            for (const Transition& e : trs) {
                auto& nv = ndp[e.key];
                if (nv.empty()) nv.assign(wid, 0);
                for (int i = 0; i < wid; i++) {
                    if (vec[i] == 0) continue; // 概率为 0 跳过
                    int ni = i + e.delta_diff; // 分差更新
                    nv[ni] += vec[i] * e.p;    // 概率累加
                }
            }
        }
        dp.swap(ndp);
    }

    // ---- 计算期望 E(|L-R|) ----
    LD ans = 0;
    for (auto& [key, vec] : dp) {
        for (int i = 0; i < wid; i++) {
            int d = i - off;       // 还原真实分差
            ans += fabsl((LD)d) * vec[i]; // |d| * probability 累加
        }
    }

    auto end = chrono::steady_clock::now();
    chrono::duration<double> sec = end - beg;

    cout << fixed << setprecision(8) << (double)ans << "\n";
    cerr << "time = " << fixed << setprecision(6) << sec.count() << "s\n";
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int T = 1;
    while (T--) {
        solve();
    }
    return 0;
}
```

## 复杂度分析

设 $S$ 为规范化后的可达状态数（实际约数百量级），$T = 50$ 为轮数，$N = 10$ 为数字种类。

每轮对每个状态枚举不超过 $N+1$ 条转移，每条转移需遍历分差数组长度 $2T+1$。

时间复杂度：$O(T \cdot S \cdot N \cdot T) = O(SNT^2)$，在本题规模下极小。

空间复杂度：$O(S \cdot T)$，存储每个状态的分差概率分布。
